# 成果物管理の3ステップフロー

## なぜ3ステップか

成果物を作っただけでは「配信済み」にならない。以下の3つが全て完了して初めて成果物が活用可能になる。

## Step 1: ローカル保存

```
workspace/output/YYYY-MM-DD/{ファイル名}
```

- 日付フォルダで整理
- ファイル名はkebab-case（例: `sales-report-march.pdf`）
- 同じ日の成果物は同じフォルダに入れる
- ファイルのバージョン管理はしない（上書き更新）

## Step 2: Notion Knowledge DBに記録

成果物のメタデータをNotionのKnowledge DBに登録する。

| プロパティ | 値の例 |
|-----------|-------|
| 名前 | 営業レポート3月 |
| 種類 | レポート / 営業資料 / コンテンツ / 提案書 / 分析 / テンプレート |
| カテゴリ | 営業 / コンテンツ / 分析 / 組織運営 |
| 担当 | エージェントID |
| タグ | 関連するタグ全て |
| ファイルパス | フルパス |
| 内容 | 概要（何を・なぜ・結果） |
| ステータス | 作成済 → レビュー中 → 配信済 / 要修正 |

## Step 3: Discord通知（Notion URL必須）

```
📦 **成果物完成**
タイトル: {名前}
内容: {概要}
Notion: {Notion APIレスポンスのURL}
```

**Notion URLが必須の理由**: スマホからワンタップでNotionページを開ける形にする。ファイルパスだけだとスマホからアクセスできない。

## 成果物の検索性

- **Notion Knowledge DB** でタグ・カテゴリ・日付で検索可能
- **Discord #feed** で時系列の通知が残る
- **ローカルファイル** は日付フォルダで整理されている

3つの経路から成果物にたどり着ける。

## よくある失敗

### Notion記録を忘れる
ローカルに保存してDiscordに通知したが、Notionに記録しなかった。後から「あの資料どこ？」と探すとき見つからない。

### Discord通知にNotion URLを入れ忘れる
テキストだけの通知だとスマホから成果物にアクセスできない。URLは必須。

### ステータスを更新しない
「作成済」のままで「配信済」に更新しない。どの成果物が最終版か分からなくなる。

## 教訓

- **「作ったら終わり」ではない**。保存 → 記録 → 通知の3ステップが完了して初めて配信済み
- ステータス管理をサボると「最新版はどこ？」問題が必ず起きる
- Notion URL（スマホからアクセス可能）の重要性は思った以上に高い
