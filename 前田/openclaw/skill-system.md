# OpenClawスキルシステムの設計と作り方

## スキルとは

エージェントが呼び出せる機能モジュール。SKILL.mdファイルで定義する。

```
skills/
├── agent-browser/SKILL.md     # ブラウザ自動化
├── slide-generator/SKILL.md   # プレゼン生成
├── sns-content-generator/SKILL.md  # SNSコンテンツ生成
├── deep-research/SKILL.md     # 深層リサーチ
└── ...（40+スキル）
```

## SKILL.mdの構造

```markdown
---
name: skill-name
description: このスキルが何をするか（1行）
read_when: どういう指示で起動するか
metadata:
  version: 1.0
  author: agent-id
---

## 概要
（スキルの説明）

## 手順
（AIが実行するステップ）

## 入力
（何を受け取るか）

## 出力
（何を出力するか）
```

## スキルの自動検出

- `skills/` ディレクトリにSKILL.mdを置くと自動的にロードされる（watchモード）
- フロントマター（name, description, read_when）は常に読まれる（トリガー判定用）
- 本文はトリガーされた時だけロードされる（コンテキスト節約）

## スキル作成のポイント

### 1. 指示は簡潔に

```
❌ 「まず〇〇を確認し、次に△△を実行し、その結果を元に...」（長文プロンプト）
✅ 「Step 1: 〇〇  Step 2: △△  Step 3: □□」（箇条書き）
```

AIは長文より構造化された指示の方が正確に実行できる。

### 2. 適度な自由度を残す

```
❌ 「フォントは14px、色は#333、マージンは16px」（全部指定）
✅ 「読みやすいフォーマットで出力」（方針だけ指定）
```

過剰に指定すると、文脈に合わない出力になる。

### 3. スクリプトを同梱できる

スキルにシェルスクリプトやNode.jsスクリプトを同梱して、AI + スクリプトのハイブリッド実行が可能。

## 代表的なスキルパターン

### リサーチ型（deep-research）
```
入力: 調査テーマ
処理: Web検索 → 情報収集 → 構造化 → レポート作成
出力: Markdownレポート
```

### 生成型（slide-generator）
```
入力: URL / PDF / テキスト
処理: ソース分析 → 構造化 → スライド生成 → エクスポート
出力: PowerPoint / PDF / reveal.js
テンプレート: corporate / pitch / training / minimal / dark
```

### 自動化型（email-automation）
```
入力: ルール定義
処理: メール受信 → 分類 → ルール適用 → アクション実行
出力: 処理結果ログ
```

## スキルの監査結果（実運用での気づき）

41スキル中：
- **完全実装**: 6スキル（15%）
- **部分実装**: 9スキル（22%）
- **ドキュメントのみ**: 23スキル（56%）
- **ドキュメントなし**: 2スキル（5%）

**教訓**: スキルを作りすぎると管理が破綻する。「ドキュメントだけのスキル」が大量に残る。実際に使うスキルだけを実装する方が効果的。
