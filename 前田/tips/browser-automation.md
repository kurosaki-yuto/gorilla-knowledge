# ブラウザ自動化で詰まるポイントと対処法

## 1. ref（要素参照）は揮発性

ブラウザツールが返す `@e1`, `@e2` のような要素参照は、**ページ遷移やDOMの更新で無効になる**。

```
❌ ページ遷移 → 古いrefでクリック → エラー
✅ ページ遷移 → snapshot取り直し → 新しいrefでクリック
```

**ルール**: ナビゲーション後は必ず `snapshot` を実行してからアクションする。

## 2. セレクタよりセマンティックロケーター

CSSセレクタはサイト更新で壊れやすい。可能ならセマンティックロケーター（役割ベース）を使う：

```
❌ document.querySelector('.btn-primary-v2')
✅ find role button --name "送信"
```

## 3. ヘッドレスブラウザの接続失敗

ブラウザへの接続がタイムアウトする場合：
- **リトライしても無駄**。Gateway（またはブラウザプロセス）の再起動が必要
- `snapshot -i`（インタラクティブ要素のみ）を使うとレスポンスが速い

## 4. セッション分離

並列でブラウザ操作する場合、セッションを分ける：
```
--session login-flow   # ログインフロー用
--session data-scrape  # データ取得用
```
セッションを分けないと、片方のナビゲーションがもう片方に影響する。

## 5. 認証の永続化

ログイン状態を保存して再利用する：
```
state save --name "my-login"   # ログイン後に保存
state load --name "my-login"   # 次回セッションで読み込み
```
毎回ログインし直すのは遅いし、レートリミットに引っかかる原因にもなる。
