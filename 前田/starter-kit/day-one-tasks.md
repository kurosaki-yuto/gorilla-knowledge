# 初日にやるべきこと

## 環境構築後のファーストステップ

環境構築が完了したら、以下の順序でシステムに慣れる。

---

## 1. Claude Code で自己紹介する（5分）

```
「自分は[名前]。[職種/役割]で、主に[技術スタック]を使う。
 [得意なこと]と[苦手なこと/学びたいこと]を覚えておいて。」
```

**例:**
```
「自分は黒崎。バックエンドエンジニアで、TypeScriptとPythonを使う。
 API設計が得意で、フロントエンドは勉強中。覚えておいて。」
```

これにより、Claude Codeがあなたに最適化された提案をするようになる。

## 2. gorilla-knowledge を読む（15分）

```bash
# リポジトリをクローン
git clone https://github.com/kurosaki-yuto/gorilla-knowledge.git
cd gorilla-knowledge

# 自分のディレクトリを確認
ls [自分の名前]/

# 前田のナレッジを読む（特にfailures/）
ls 前田/
```

**特に読むべきファイル:**
- `前田/failures/` — 全ファイル（同じ失敗をしないため）
- `前田/claude-code/effective-prompting.md` — 指示の出し方
- `前田/claude-code/claude-md-design.md` — CLAUDE.mdの書き方

## 3. 最初のナレッジを書く（15分）

```bash
cd gorilla-knowledge

# 自分のディレクトリで作業
cd [自分の名前]/

# 環境構築でつまずいたことを記録
# 例: vim setup-notes.md
```

**書くこと:**
- 環境構築でハマったポイント
- 解決方法
- 「最初に知りたかった」こと

```bash
# コミット＆プッシュ
git pull origin main
git add [自分の名前]/
git commit -m "環境構築メモを追加"
git push origin main
```

## 4. Discord で挨拶する（1分）

#main チャンネルで「環境構築完了しました」と投稿。Bot が反応すれば接続もOK。

## 5. 小さなタスクをやってみる（30分）

Claude Code に簡単なタスクを頼んで、ワークフローを体験する：

```
「このプロジェクトの README.md を読んで、内容を要約して」
```

```
「[自分のディレクトリ]に、今日学んだことをまとめるファイルを作って」
```

```
「src/ ディレクトリの構造を調べて、主要なファイルの役割を説明して」
```

---

## やらないこと（初日）

- ❌ システム設定を変更する（まず全体像を把握してから）
- ❌ 大きな機能を実装する（まずワークフローに慣れてから）
- ❌ 他のメンバーのディレクトリを編集する（自分のディレクトリのみ）

| 日 | 目標 |
