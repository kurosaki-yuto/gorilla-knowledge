# ツール実行時のエラーパターンと対処法

## 概要

AIエージェントがツールを呼び出す際、特定のパターンでエラーが発生する。ここでは実運用で遭遇したエラーと、効果的だった対処法をまとめる。

## パターン1: ファイル操作エラー（ENOENT / EISDIR）

**症状**: ファイルの読み書きで「ファイルが見つからない」または「ディレクトリをファイルとして扱おうとした」エラー

**対処**: ツールのフォールバックチェーンを使う
```
read（ファイル読み込み） → exec（シェルコマンド） → browser（ブラウザ経由）
```
1つ目のツールが失敗しても、別のツールで同じ操作を試みると成功することが多い。

## パターン2: ブラウザ自動化の接続失敗

**症状**: ヘッドレスブラウザへの接続がタイムアウト

**対処**:
- **単純なリトライでは解決しない**。Gatewayの再起動が必要
- セレクタのミスマッチは `snapshot` コマンドで現在のDOM状態を再取得してから操作する
- ナビゲーション後は必ずスナップショットを取り直す（古いref IDは無効になる）

## パターン3: 外部API取得の失敗（503 / セキュリティブロック）

**症状**: `web_fetch` で外部URLを取得しようとすると503やセキュリティ警告

**対処**:
- `web_fetch` → `browser`（ブラウザツール）にフォールバック
- ブラウザ経由だとセキュリティブロックを回避できることが多い

## パターン4: テキスト編集の不一致

**症状**: ファイル編集ツールで「指定したテキストが見つからない」エラー

**対処**:
- **編集前に必ずファイルを読む**。空白・インデント・改行が1文字でもズレると失敗する
- コピペではなく、readの結果をそのまま使う

## パターン5: Discordマルチチャンネルのルーティング失敗

**症状**: `guildId required` や `Unknown channel` エラー

**対処**:
- 複数チャンネルが設定されている場合、暗黙のルーティングは失敗する
- メッセージ送信時に **明示的にチャンネルIDを指定** する必要がある

## 全体の教訓

- **1つのツールに固執しない**。フォールバックチェーンを持つ
- **リトライで解決しない問題はリトライしない**（ブラウザ→Gateway再起動など、別の対処が必要）
- **外部サービスのエラーは設定を疑う**（権限、Intent、共有設定etc）
