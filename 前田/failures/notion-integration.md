# Notion API連携で全DBが404になった話

## 何が起きたか

Notion APIを使ってDB（Goals, Projects, Commands, Knowledge, Confirmation）にアクセスしようとしたら、全てのDBで `object_not_found` エラーが返ってきた。

```
Error: Could not find object with ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

APIトークンは正常（認証OK）。接続テストも通る。なのにDB操作だけ全滅。

## 原因

**Notionインテグレーションが各DBに「共有」されていなかった。**

Notion APIでDBにアクセスするには：
1. インテグレーションを作成する ✅
2. APIトークンを取得する ✅
3. **各DBの「接続を追加」でインテグレーションを選択する** ← これが抜けていた

つまり、トークンが有効でも、DBごとに明示的にアクセス権を付与しないと `object_not_found` になる。

## 教訓

- **Notion APIはDB単位でアクセス権が必要**。ワークスペース全体に自動適用されない
- トークンが通る ≠ DB にアクセスできる。この2つは別レイヤー
- エラーメッセージが `object_not_found`（403ではない）なので、「IDが間違ってる？」と誤認しやすい
- 新しいDBを作ったら、毎回「接続を追加」を忘れずにやる

## 修正方法

```
Notion UI → 対象DB → 右上「...」→ 「接続を追加」→ インテグレーション名を選択
```

全DBに対して1つずつやる必要がある（一括設定は不可）。

## 追加で発覚した問題

自動DB作成スクリプト（起動時に自動初期化）も未実装だった。ドキュメントには「起動時に自動作成」と書いてあったが、実装されていなかった。ドキュメントと実装の乖離チェックも大事。
