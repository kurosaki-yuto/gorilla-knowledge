# コンテキストウィンドウを起動時に8.7%消費していた問題

## 何が起きたか

AIエージェントのセッション起動が遅く、すぐにコンテキストが溢れる問題が発生。調査したところ、セッション開始時点で約17,400トークン（200kコンテキストの8.7%）を消費していた。

## 原因の内訳

| ファイル | サイズ | トークン数 |
|---------|--------|-----------|
| SOUL.md（エージェント定義） | 35KB | ~9,000 |
| AGENTS.md | 8KB | ~2,000 |
| USER.md | 4KB | ~1,100 |
| デイリーログ x2 | 6KB | ~1,500 |
| その他設定ファイル | - | ~3,800 |
| **合計** | - | **~17,400** |

特にSOUL.md（35KB）が最大の原因。1ファイルでコンテキストの4.5%を食っていた。

さらに別の問題として、会議の文字起こしログ（916KB、1ファイル122KB）がメモリに読み込まれると、それだけで15%を消費していた。

## 教訓

- **起動時に読み込むファイルは小さく保つ**。8KB以下を目標にする
- 巨大なプロンプト定義は「起動時に全部読む」のではなく、必要な部分だけ遅延読み込みする
- ログファイルは要約版（summary.md）と生データ（raw/）に分離する
- 7日以上前のログは自動アーカイブする仕組みを入れる
- メモリの断片化（3箇所に分散）も問題。1箇所に集約すべき

## 設計した改善策

```
memory/
├── daily/          # 当日〜7日分のログ
├── archive/        # 8日以上前（必要時のみ読む）
├── SOUL.md         # 8KB以下に圧縮（95%削減）
└── summary.md      # 長文ログの要約版
```

目標: 起動時コストを17,400 → 5,000トークン以下（-71%）に削減
