# Gatewayが毎分偽アラートを出し続けた問題

## 何が起きたか

OpenClaw Gateway の監視機能が「15分間活動がありません」というアラートを**毎分**送信し続けた。実際にはエージェントは正常に動作していた。

## 原因

Gateway内部のモニタリング関数がハードコードされており、`openclaw.json`（外部設定ファイル）からは制御できなかった。

- 閾値の変更不可
- アラート頻度の調整不可
- アラートの無効化不可

## 教訓

- **外部から設定変更できない機能は運用上の地雷になる**
- アラートの頻度制限（レートリミット）は必須。同じアラートを毎分送るのはノイズでしかない
- 監視システムを導入する前に「どの設定が外部から変更可能か」を確認する
- フレームワーク・プラットフォームの制約は、使い始める前に把握すべき

## 対処

暫定対処として、受信側（CEOエージェント）でこのアラートをフィルタリングして無視するようにした。根本解決にはGatewayのソースコード修正かフィーチャーリクエストが必要。
