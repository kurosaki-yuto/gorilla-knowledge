# エージェント間コミュニケーション設計

## Discordチャンネル設計

エージェントシステムではDiscordをコミュニケーション基盤として使う。チャンネルの役割を明確に分離する：

```
#main    → ユーザー ↔ CEO の対話チャンネル
#feed    → 成果物の通知（Notion URL必須）
#alert   → 異常時のアラートのみ（Ops Monitorが管理）
#log     → マシンログ（デバッグ・監査用、読み専用）
```

### 設計原則

- **#main はCEOだけが書く**。サブエージェントはCEOに報告し、CEOがユーザーに伝える
- これにより、ユーザーのアテンション（注意力）を守る
- 複数エージェントが同時にユーザーに話しかけると混乱する

### スレッドの活用

- チャンネル = チーム単位
- スレッド = タスク単位
- 1タスク1スレッドで文脈を分離する

## 「沈黙 = 正常」の原則

Ops Monitorは正常時に何も通知しない。

```
✅ 全チェック正常 → [何もしない]
⚠️ 警告レベル → 自動修復を試行 → #log に記録
🔴 致命的 → #alert に通知 + CEOに報告
```

**なぜ重要か**: 「異常なし」の報告を毎日受けると、それがノイズになって本当のアラートを見逃す。

## 水平連携（v7での進化）

v7から「共有ボード」を導入。明示的なハンドオフなしで、エージェント間の情報共有を実現：

### プロジェクトボード
- 案件ごとのタスクと成果物を記録
- 各エージェントが自分の担当部分を書き込む
- 例: A社提案 → リサーチャーが市場調査結果を書く → セールスがそれを読んで提案書を作る

### インテリジェンスボード
- 市場データ、政策動向、顧客フィードバック、SNSトレンドを蓄積
- `_index.md` で「何が新しいか」を管理
- エージェントが定期的にチェックして、関連する情報を発見する

### ハートビート頻度

| エージェント | チェック頻度 | 目的 |
|-------------|------------|------|
| Kevin（CoS） | 3時間 | 水平パスの調整 |
| Mike/Jake/Ryan | 4時間 | 新情報の発見 |
| Marcus（Ops） | 6時間 | 異常検知 |

**ポイント**: 明示的な「ハンドオフ」がなくても、ボードに情報を書けば他のエージェントが拾える。
