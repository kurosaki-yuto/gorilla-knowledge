# 実用スキルカタログ — 実際に使えたスキルとその活用法

## スキルとは

Claude Codeに「こういう手順でこういう作業をやって」と教え込む定型化された指示セット。一度作れば何度でも再利用できる。

---

## Tier 1: すぐ使える即効スキル

### slide-generator（プレゼン自動生成）

**何ができるか**: URL、PDF、テキストからスライドを自動生成

**使い方**:
```
「このURLの内容をもとに、10枚のプレゼン資料を作って」
「このPDFを要約してピッチデックにして」
```

**出力形式**: PowerPoint / PDF / reveal.js / Google Slides

**テンプレート**: corporate / pitch / training / minimal / dark

**実用例**:
- 競合分析レポートからプレゼンを自動生成
- ブログ記事をセミナー資料に変換
- 技術ドキュメントを社内共有用スライドに

**注意点**: 生成後に人間がレビュー・微調整する前提。100%完成品が出るわけではない

---

### deep-research（深層リサーチ）

**何ができるか**: テーマを渡すとWeb検索→情報収集→構造化レポート

**使い方**:
```
「〇〇について調査して、Markdownレポートにまとめて」
```

**仕組み**:
1. ブラウザ自動化でGemini/ChatGPTのDeep Research機能を呼び出す
2. 調査完了をポーリングで待つ（5分初回待ち → 60秒間隔チェック → 20分タイムアウト）
3. 結果を抽出してMarkdownに整形

**レート制限**:
- Gemini: 1日5回まで（無料枠）
- ChatGPT: 月間制限あり
- 使用回数をJSONで追跡して超過を防止

**実用例**:
- 技術選定の比較調査
- 市場調査・競合分析
- 最新トレンドの定期レポート

---

### email-automation（メール自動振り分け）

**何ができるか**: 未読メールを自動分類してアクション実行

**分類ルール**:

| 分類 | アクション |
|------|-----------|
| URGENT（緊急） | スター付け + 通知 |
| NEWSLETTER | 自動アーカイブ |
| RECEIPT（領収書） | ラベル付け + アーカイブ |
| SPAM | ログ記録のみ |
| OTHER | 何もしない |

**学習機能**: 分類結果をメモリに保存。次回以降の分類精度が上がる

## Tier 2: 業務に合わせて活用するスキル

### calendar-briefing（朝ブリーフィング）

**何ができるか**: Googleカレンダーの予定を分析して朝のブリーフィングを自動生成

**出力例**:
```
## 今日の予定（3月19日）
- 10:00 チームMTG（1時間）
- 14:00 クライアントA打ち合わせ（30分）
- フォーカスタイム: 午前中2時間、午後3時間

## 優先タスク
- プレゼン資料の最終確認（明日期限）
- コードレビュー2件
```

**Cron設定**: 毎朝5時に実行 → Slack/Discordに投稿

---

### agent-browser（ブラウザ自動操作）

**何ができるか**: ヘッドレスブラウザでWebサイトを自動操作

**使い方**:
```bash
agent-browser open https://example.com    # ページを開く
agent-browser snapshot -i                  # 操作可能な要素を取得
agent-browser click @e1                    # 要素をクリック
agent-browser fill @e2 "入力テキスト"       # フォーム入力
agent-browser screenshot                   # スクリーンショット
```

**実用例**:
- Webサイトのスクレイピング
- フォームの自動入力
- UIテストの自動化
- ログイン後のデータ取得（セッション保存可能）

---

### sns-content-generator（SNSコンテンツ一括生成）

**何ができるか**: テーマを渡すとInstagram/X向けのコンテンツを一括生成

**出力**:
- キャプション（本文）
- ハッシュタグ
- 画像の提案
- 投稿推奨時間

## Tier 3: 自分でスキルを作る

### skill-creatorのポイント

新しいスキルを作るときの指針：

1. **コンテキストは公共財**。SKILL.mdは短く書く
2. **構造化された手順** > 長文の説明
3. **適度な自由度を残す**。全部指定すると文脈に合わない出力になる
4. **スクリプトを同梱できる**。決定的な処理はスクリプトでやる

### 最小構成のスキル

```markdown
---
name: my-skill
description: ○○を自動化するスキル
---

## 手順
1. ○○を確認する
2. △△を実行する
3. 結果を□□に保存する
```

これだけで動く。まず小さく作って、必要に応じて拡張する。

### スキル監査の教訓

41スキル作った結果：
- 実際に使ったのは **6スキル**（15%）
- 残りはドキュメントだけ or 部分実装

**教訓**: 「いつか使うかも」で作らない。今使うものだけ作る。
