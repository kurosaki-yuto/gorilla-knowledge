# エージェント・スキルの追加手順チェックリスト

## エージェント追加時の6ステップ

新しいエージェントを追加するときは、以下を全て完了させる。1つでも抜けると不整合が起きる。

### Step 1: 設定ファイルに定義を追加

`agents.yaml`（または同等の設定ファイル）にエージェント定義を追加：
- エージェントID
- 使用モデル
- 担当する責務（1エージェント = 1責務）
- 所属する層（戦略 / 調整 / 実行）

### Step 2: エージェントのプロンプト定義

`SOUL.md`（システムプロンプト）を作成：
- 役割の明確な定義
- **できること / できないこと** を明示的に書く
- 他のエージェントとの連携方法
- 8KB以下に収める（コンテキスト節約）

### Step 3: ワークスペースフォルダを作成

エージェント専用の作業ディレクトリを用意する。

### Step 4: ツール定義

`TOOLS.md` にエージェントが使えるツールを定義。不要なツールは明示的に禁止する。

### Step 5: ルーティング設定

`routing.yaml`（または同等の設定）にメッセージのルーティングルールを追加。どのチャンネル/トリガーでこのエージェントが起動するかを定義。

### Step 6: 設計ドキュメントを更新

システム設計書にエージェントの追加を反映。他のエージェントとの関係図も更新する。

## スキル追加時の手順

スキル（エージェントが呼び出せる機能モジュール）の追加：

1. `skills/{skill-name}/SKILL.md` にスキル定義を作成
2. トリガー条件を明記（どういう指示でこのスキルが起動するか）
3. 入力/出力のフォーマットを定義
4. エージェント設定でスキルの使用を許可

## よくある失敗

- **SOUL.mdが巨大になる**: 起動時コンテキストを圧迫。8KB以下を死守
- **ツール定義を忘れる**: エージェントが何もできない状態になる
- **ルーティングを忘れる**: エージェントは存在するが、メッセージが届かない
- **役割の重複**: 2つのエージェントが同じことをやり始めて混乱する
