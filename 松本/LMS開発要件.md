# 自社LMS 開発要件

> 最終更新: 2026年3月19日
> 背景: 助成金（人材開発支援助成金 定額制訓練）に対応するLMSを自社開発する

---

## 何を作るか

助成金申請に必要な証跡を出力できるeラーニングプラットフォーム。
企業がサブスク契約 → 社員がLMSで動画研修を受講 → 受講記録・修了証を出力 → 企業が助成金申請に使う。

---

## コア機能（5つ）

### 1. 動画プレイヤー（視聴ログ付き）

```
必須:
  ・動画の再生/停止
  ・視聴開始日時・完了日時の記録（DB保存）
  ・動画ごとの「標準学習時間」を表示（例: 20分）
  ・視聴完了の判定（最後まで再生したかどうか）

推奨:
  ・初回視聴時の早送り/スキップ禁止
  ・途中再開（前回の続きから再生）
```

### 2. テスト機能

```
必須:
  ・選択式テスト（3〜5問/コンテンツ）
  ・合格判定（正答率80%以上）
  ・受験日時・スコアの記録（DB保存）

※ テスト時間は助成金の時間にカウントされない
  → 時間記録は不要、合否だけ取ればOK
```

### 3. 修了判定 + 修了証PDF発行

```
修了条件:
  ・動画視聴が完了している
  ・確認テストに合格している
  → 両方満たしたら「修了」ステータスに変更

修了証PDFに記載する項目:
  ・受講者氏名
  ・コンテンツ名（コース名）
  ・標準学習時間
  ・修了日
  ・訓練機関名（うちの会社名）
  ・発行者の署名 or 印影

重要: 未修了の人には修了証が出せない制御が必須
```

### 4. 受講履歴のCSV出力

```
出力カラム:
  ・受講者名
  ・コンテンツID
  ・コンテンツ名
  ・標準学習時間（分）
  ・視聴開始日時
  ・視聴完了日時
  ・テスト合否
  ・修了ステータス（修了 / 未修了）
  ・修了日

→ 企業ごとにフィルタしてCSVダウンロードできること
```

### 5. 管理画面

```
コンテンツ管理:
  ・動画アップロード
  ・標準学習時間の設定
  ・テスト問題の登録

企業管理:
  ・企業の登録（契約情報）
  ・受講者の登録（企業に紐づく）

レポート:
  ・受講状況の一覧表示（企業別・受講者別）
  ・CSVダウンロードボタン
  ・修了証の一括/個別ダウンロード
```

---

## LMSから出力するもの（4つ）

企業が助成金申請で労働局に提出するために、うちのLMSから出力して渡すもの。

| # | 出力物 | 形式 | 内容 |
|---|--------|------|------|
| 1 | **受講履歴データ** | CSV | 誰が・いつ・何を・何分受講して・修了したか |
| 2 | **修了証** | PDF | 受講者ごとに1枚。修了者のみ発行 |
| 3 | **カリキュラム一覧** | PDF/CSV | コンテンツID・タイトル・標準学習時間の一覧 |
| 4 | **サービス利用料の領収書** | PDF | 企業が「全額負担した」ことの証拠 |

## データベース設計（最小構成）

```
companies（企業）
  ├ id
  ├ name（企業名）
  ├ contract_start_date（契約開始日）
  ├ contract_end_date（契約終了日）
  └ plan（プラン）

users（受講者）
  ├ id
  ├ company_id → companies.id
  ├ name（氏名）
  └ email

contents（コンテンツ）
  ├ id
  ├ title（タイトル）
  ├ standard_duration_min（標準学習時間・分）
  ├ video_url（動画URL）
  └ level（レベル: 入門/実践/応用/管理者）

quizzes（テスト問題）
  ├ id
  ├ content_id → contents.id
  ├ question（問題文）
  ├ choices（選択肢・JSON）
  └ correct_answer（正解）

watch_logs（視聴ログ）
  ├ id
  ├ user_id → users.id
  ├ content_id → contents.id
  ├ started_at（視聴開始日時）
  ├ completed_at（視聴完了日時・NULL=未完了）
  └ is_completed（完了フラグ）

quiz_results（テスト結果）
  ├ id
  ├ user_id → users.id
  ├ content_id → contents.id
  ├ score（得点）
  ├ passed（合否）
  └ taken_at（受験日時）

completions（修了記録）
  ├ id
  ├ user_id → users.id
  ├ content_id → contents.id
  ├ completed_at（修了日）
  └ certificate_issued（修了証発行済フラグ）
```

## 画面一覧（最小構成）

### 受講者向け

| 画面 | 内容 |
|-----|-----|
| ログイン | メール+パスワード |
| コンテンツ一覧 | 動画一覧、進捗率表示、修了済みマーク |
| 動画視聴 | プレイヤー + 標準学習時間表示 |
| 確認テスト | 選択式、合否表示、再受験可能 |
| 修了証ダウンロード | 修了済みコンテンツの修了証PDF |

### 管理者向け

| 画面 | 内容 |
|-----|-----|
| ダッシュボード | 企業別の受講状況サマリー |
| 企業管理 | 企業CRUD、契約情報 |
| 受講者管理 | 受講者CRUD（企業紐づき） |
| コンテンツ管理 | 動画アップ、標準学習時間設定、テスト登録 |
| レポート | 受講履歴一覧、CSVダウンロード |
| 修了証管理 | 一括/個別PDF発行・ダウンロード |
