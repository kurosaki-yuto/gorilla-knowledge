# Skill Hunter - 最強スキルを発掘してパクるスキル

他の人が「もっと早く知りたかった」「入れて後悔なし」と言っているClaude Codeスキルを探し出し、ユーザーの環境に最適な形でインストールする。

## ワークフロー

### Phase 1: ニーズのヒアリング

ユーザーに以下を確認する（省略された場合はデフォルトで進める）:

1. **探したいジャンル**（未指定なら「全ジャンル人気順」）
   - 開発効率化（コードレビュー、テスト、リファクタ）
   - ドキュメント・資料作成（docx, pptx, pdf）
   - フロントエンド（React, CSS, デザイン）
   - DevOps・インフラ（Docker, CI/CD, AWS）
   - AI・エージェント開発
   - その他（ユーザー指定）

2. **インストール先**
   - `~/.claude/skills/` （個人グローバル、デフォルト）
   - `.claude/skills/` （プロジェクトローカル）

3. **何個くらい入れたいか**（デフォルト: 3〜5個を厳選）

### Phase 2: スキル発掘（並列リサーチ）

以下のソースから並列でスキルを探す。**必ず複数のサブエージェントを並列起動**してリサーチする:

#### 情報ソース（優先順）

1. **awesome-claude-code** (GitHub: hesreallyhim/awesome-claude-code)
   - 最も網羅的なキュレーションリスト
   - THE_RESOURCES_TABLE.csv にスター数・カテゴリ付きで整理されている
   - URL: `https://github.com/hesreallyhim/awesome-claude-code`

2. **awesome-claude-skills** (GitHub: ComposioHQ/awesome-claude-skills)
   - 実用的なスキル集
   - URL: `https://github.com/ComposioHQ/awesome-claude-skills`

3. **Claude Code公式マーケットプレイス**
   - `/plugin marketplace` コマンド経由でインストール可能
   - alirezarezvani/claude-skills（192+スキル）
   - daymade/claude-code-skills
   - claudemarketplaces.com

4. **コミュニティの声**（日本語含む）
   - Zenn, Qiita, Dev.to, Medium の記事
   - 検索キーワード例: "Claude Code Skills おすすめ", "best claude code skills 2026", "claude skills game changer"
   - X/Twitter での反響

5. **GitHub トレンド**
   - `claude-code-skills`, `claude-skills` トピックのリポジトリ
   - スター数、最終更新日、Issue活動度で評価

#### 評価基準

各スキルを以下の基準でスコアリング:

| 基準 | 重み | 説明 |
|------|------|------|
| 人気度 | 30% | GitHub Stars、インストール数、記事での言及数 |
| 実用性 | 25% | 日常的に使える頻度、汎用性 |
| 品質 | 20% | コードの質、ドキュメント、メンテナンス状況 |
| 独自性 | 15% | 既存の組み込み機能では代替できない価値 |
| 鮮度 | 10% | 最終更新日、最新のClaude Code機能への対応 |

### Phase 3: レポート作成

調査結果を以下のフォーマットで提示する:

```
## 発掘結果レポート

### おすすめスキル TOP N

#### 1. [スキル名] ⭐ [スター数]
- **何がすごいか**: 1行で
- **みんなの声**: 実際のコメント・レビューから引用
- **ソース**: GitHub URL
- **インストール方法**: コマンド or 手動
- **既存スキルとの重複**: あり/なし（ありの場合、何と被るか）

...（繰り返し）

### 見送ったスキル（理由付き）
- [スキル名]: [見送り理由]
```

**重要**: ユーザーが既に持っているスキルと重複するものは明示する。重複チェックは以下で行う:
- `~/.claude/skills/` の既存スキル一覧
- `.claude/skills/` の既存スキル一覧
- 機能的に同等なものも検出（名前が違っても機能が被っていれば指摘）

### Phase 4: インストール

ユーザーが選んだスキルをインストールする。

#### インストール方法（優先順）

1. **公式マーケットプレイス経由**（対応している場合）
   ```
   /plugin marketplace add [marketplace-repo]
   /plugin install [skill-name]@[marketplace]
   ```

2. **GitHubからクローン**
   ```bash
   # スキルディレクトリに直接クローン
   git clone [repo-url] ~/.claude/skills/[skill-name]
   # または必要なファイルだけコピー
   ```

3. **手動作成**（小さいスキルの場合）
   - SKILL.md と必要なリソースを直接書き出す
   - ソースを参考にしつつ、ユーザーの環境に合わせてカスタマイズ

#### インストール後の処理

- SKILL.md の description を確認し、トリガー条件が適切か検証
- 既存スキルとの名前衝突がないか確認
- 簡単な動作テスト（可能であれば）

### Phase 5: カスタマイズ提案

インストールしたスキルについて:
- ユーザーの環境・ワークフローに合わせた調整提案
- 日本語化が必要なら対応
- 不要な依存関係の削除

## 注意事項

- **ライセンス確認**: インストール前に必ずライセンスを確認。MIT/Apache-2.0 なら問題なし。
- **セキュリティ**: スキル内に不審なBashコマンドやAPIキー送信がないか確認。
- **最小限主義**: 「とりあえず全部入れる」ではなく、本当に使うものだけ厳選する。大量にインストールするとコンテキストが膨らむ。
- **既存スキルの尊重**: ユーザーが既にカスタマイズしているスキルを上書きしない。
