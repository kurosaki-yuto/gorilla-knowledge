# 定額制AI研修 制作ロードマップ（助成金通過設計）

> 最終更新: 2026年3月19日

---

## 最初に理解すべきこと：助成金が通る構造

### 定額制訓練で助成金を通すには「商品の形」が決まっている

```
助成金が通る構造:

  企業（受講側）
    ↓ サブスク契約（月額 or 年額）
  訓練機関（うち）← ここの要件を満たす必要がある
    ↓ LMSでコンテンツ提供
  受講者（企業の社員）
    ↓ 受講記録がLMSに残る
  企業が労働局に助成金申請
```

### 超重要：クリアすべき5つの条件

```
① コンテンツは「特定の事業主向け」であってはならない
   → 汎用的なAI研修として複数企業に提供する形にする
   → 1社専用のカスタムコンテンツは助成対象外

② 定額制サービス（サブスク型）として提供する
   → 月額 or 年額の定額で、複数コンテンツが見放題の形
   → 1講座いくらの従量課金ではダメ

③ LMSで受講記録を取れること
   → 誰が・いつ・何を・何分受講したかの記録
   → 修了証の自動発行機能
   → 受講時間（動画視聴時間）の記録

④ 各受講者が1時間以上受講していること
   → 1時間未満の人はカウント対象外

⑤ 全対象者の合計受講時間が10時間以上であること
```

## Phase 0: 事業基盤の整備（最優先・コンテンツ制作より先）

コンテンツを作る前に、**助成金が通る器（箱）を先に作る**。

### 0-1. LMSの選定・構築

助成金審査に耐えるLMSの必須機能：

| 機能 | 必須/推奨 | 理由 |
|-----|----------|------|
| 受講者別の視聴時間記録 | **必須** | 助成金申請時に提出する |
| 修了判定（動画視聴完了 or テスト合格） | **必須** | 修了証明に必要 |
| 修了証の自動発行 | **必須** | 未修了者には発行しない制御が必要 |
| 受講履歴のCSV/PDF出力 | **必須** | 企業が労働局に提出する |
| 進捗率の表示 | **必須** | 受講管理のため |
| 初回視聴の早送り防止 | **推奨** | 不正受給防止の証拠になる |
| 確認テスト機能 | **推奨** | 修了判定の根拠強化 |
| 標準学習時間の表示 | **必須** | 助成金の時間カウントの根拠 |

**既存LMSを使うのが最速。** 自前で作ると審査で不安要素になる。

候補例：
- learningBOX
- AirCourse
- manebi
- Schoo for Business
- UMU

→ **「助成金対応」を謳っているLMSを選ぶのが鉄則**

### 0-2. サービス体系の設計

```
【定額制サービスとしての商品設計】

サービス名:  （例）「AI活用リテラシー研修パック」
提供形態:    定額制（サブスクリプション）
契約単位:    月額 or 年額 × 利用人数
価格例:      月額5,000円/人（年額60,000円/人）

含まれるもの:
  ├ AI研修動画コンテンツ（全18本・計6時間分）見放題
  ├ 各コンテンツの確認テスト
  ├ 修了証の自動発行
  └ 受講履歴レポートの提供（CSV/PDF）
```

### 0-3.「特定事業主向けでない」条件のクリア

**ここが一番の落とし穴。**

```
NG: A社専用にカスタマイズしたAI研修
NG: A社の業務フローに特化した内容
NG: A社の社名やロゴが入ったコンテンツ

OK: 「AI活用」という汎用テーマで、どの会社でも使える内容
OK: 複数企業に同じコンテンツを提供する
OK: 汎用コンテンツ50〜70% + 業界共通の活用事例30〜50%
```

**対策:**
- コンテンツは **汎用的な内容** で作る（特定企業名を出さない）
- **複数の企業に提供可能な形** にする（1社専用にしない）
- Webサイト等で **一般向けに公開・販売** していることが望ましい
- サービス紹介ページを作って「誰でも契約できる」状態にしておく

### 0-4. 費用還流の防止（不正受給リスク対策）

日経新聞の報道で **助成金の3割が不適切** と指摘された。主な手口：

```
NG: 研修費を請求 → 企業に「業務協力費」として一部返金
NG: 「実質無料で受けられます」と営業
NG: 助成金額と返金額が一致するキックバック
```

**対策:**
- 値引き・キックバック・紹介料は一切やらない
- 価格は助成金の有無に関わらず一律にする
- 契約書に「費用の還流は行わない」旨を明記

## Phase 1: コンテンツ設計（助成金要件を満たす設計）

### 各コンテンツに必要なメタデータ

```
各コンテンツ（20分動画）に必ず設定するもの:

├── コンテンツID（例: AI-101）
├── タイトル
├── 標準学習時間: 20分  ← 助成金の時間カウントに使われる
├── 対象者レベル（入門/実践/応用）
├── 学習目標（3つ程度）
├── カリキュラム概要（スライドごとの内容）
├── 講師情報（氏名・経歴・専門分野）
├── 確認テスト（3〜5問・合格基準: 80%）
└── 修了条件（動画視聴完了 + テスト合格）
```

### 講師要件

```
助成金の要件:
  「訓練指導員免許を有するもの、または
   当該科目の内容について専門的な知識・技能を持つ講師」

→ AI研修の場合:
  ・AI/DX関連の実務経験
  ・関連資格（G検定、DS検定、AWS認定等）
  ・IT系の教育・研修の実績

必要書類:
  ・講師の経歴書（職歴・専門分野）
  ・保有資格の証明書
  ・過去の研修実績一覧

※ AI音声ナレーションの場合でも「監修者」として
  専門知識を持つ人物を立てる必要がある
```

## Phase 2: コンテンツ制作

### 1本あたりの成果物

```
各コンテンツ（20分）:

├── スライド（15〜20枚）
├── ナレーション台本
├── 動画（約20分）
├── 確認テスト（3〜5問・選択式）
├── 講師プロフィール
└── 学習目標・カリキュラム概要（LMS登録用）
```

### 制作フロー

```
STEP 1: 構成+台本+テスト一括生成（Claude）  … 15分
STEP 2: スライド制作（Gamma等）             … 15分
STEP 3: 音声生成（AI音声ツール）            … 15分
STEP 4: 動画化（スライド+音声合成）          … 15分
STEP 5: LMS登録用メタデータ作成             … 10分
STEP 6: レビュー・修正                      … 20分

合計: 約1.5時間/本 → 全18本で約27時間（3〜4日）
```

## Phase 3: LMS登録・動作確認

```
□ 全18本をLMSにアップロード
□ 各コンテンツの標準学習時間が正しく設定されているか確認
□ 確認テストが正常に動作するか確認
□ 修了判定（視聴完了+テスト合格）が正しく動くか確認
□ 修了証が自動発行されるか確認
□ 受講履歴のCSV出力が正しいか確認
□ テスト受講者で一連の流れを通しテスト
```

## Phase 4: 販売・提供体制の整備

### 助成金が通るために「サービスとして公開されている」状態を作る

```
□ サービス紹介ページ（LP）を作る
  └ サービス名・内容・価格・申込方法を明記
  └ 「どの企業でも契約可能」であることを示す

□ 利用規約・契約書テンプレートを作る
  └ 定額制サービスとしての契約条件
  └ 費用還流を行わない旨の明記

□ 受講案内テンプレートを作る
  └ 各コンテンツの標準学習時間一覧
  └ 修了条件の説明
  └ LMSのログイン方法

□ 助成金申請サポート資料を作る
  └ 計画届の記載例
  └ 「どの書類に何を書けばいいか」のガイド
  └ 社労士への橋渡し体制
```

## 全体スケジュール

```
【Week 1】事業基盤
  Day 1-2: LMS選定・契約
  Day 3:   サービス体系・価格設計の確定
  Day 4:   講師プロフィール・経歴書の準備
  Day 5:   LP・利用規約のドラフト

【Week 2】コンテンツ制作
  Day 1:   テンプレ確定 + レベル1（4本）制作
  Day 2:   レベル2（5本）制作
  Day 3:   レベル3（5本）制作
  Day 4:   レベル4（4本）制作
  Day 5:   全体レビュー・修正

【Week 3】公開準備
  Day 1-2: LMS登録・動作確認
  Day 3:   LP公開・利用規約確定
  Day 4:   助成金申請サポート資料作成
  Day 5:   テスト運用・最終チェック

→ 最速3週間で販売開始
```
