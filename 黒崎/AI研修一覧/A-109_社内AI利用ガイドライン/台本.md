**【ナレーション】**
この動画では、社内でAIを安全かつ効果的に使うために欠かせない「AI利用ガイドライン」について解説します。AIツールが急速に普及する中、ルールなしに使い始めると思わぬリスクを招くことがあります。ガイドラインがなぜ必要で、どんな内容を盛り込むべきなのか。20分で一緒に学んでいきましょう。

## スライド 3: AIが職場に浸透する今（100秒）

**【スライド内容】**
- 見出し: AIが職場に浸透する今
- 統計データ:
  - 2025年時点でビジネスパーソンの約7割がAIツールを業務利用
  - 社内ルール未整備の企業は約4割
- 図解: AIツール活用の広がり（メール・資料作成・コード生成・データ分析）
- キーメッセージ: 「使う人は増えているが、ルールが追いついていない」

**【ナレーション】**
まず現状を確認しましょう。ChatGPTやClaude、Copilotといった生成AIツールが急速に普及し、2025年時点でビジネスパーソンの約7割が何らかのAIツールを業務で使っていると言われています。メールの下書き、資料作成、プログラミング補助、データ分析。用途はどんどん広がっています。ところが、社内でAI利用のルールを明確に定めている企業はまだ約6割にとどまります。つまり残りの4割は、ルールがないまま社員が各自の判断でAIを使っている状態です。これは言わば、交通ルールがない道路を車が走っているようなもの。事故が起きるのは時間の問題です。だからこそ、今ガイドラインを整備することが急務なのです。

## スライド 4: ガイドラインが必要な3つの理由（100秒）

**【スライド内容】**
- 見出し: ガイドラインが必要な3つの理由
- 3つのカード:
  1. 情報漏えいリスクの防止 — 機密情報や個人情報の外部流出を防ぐ
  2. 品質・正確性の担保 — AIの誤出力による業務ミスを防ぐ
  3. 法的・倫理的リスクの回避 — 著作権侵害や差別的出力を防ぐ

**【ナレーション】**
では、なぜガイドラインが必要なのか。大きく3つの理由があります。1つ目は情報漏えいリスクの防止です。社員が外部のAIサービスに顧客の個人情報や社内の機密データを入力してしまうと、そのデータがAIの学習に使われたり、第三者に漏れる可能性があります。実際に、イタリアではChatGPTがGDPR違反として一時的に利用禁止になりました。また、ある金融機関では社員がAIチャットボットに顧客の口座情報を入力し、コンプライアンス違反として処分を受けた事例もあります。2つ目は品質と正確性の担保です。AIはもっともらしいが間違った情報を出力することがあります。ハルシネーションと呼ばれるこの現象を知らずに、AIの出力をそのまま取引先に送ってしまうと、会社の信用を損ないます。3つ目は法的・倫理的リスクの回避です。AIが生成したコンテンツには著作権の問題があり、また入力内容によっては差別的な出力がなされる可能性もあります。これら3つのリスクに組織として対処するために、ガイドラインが不可欠なのです。

## スライド 5: 主要項目① 利用範囲の定義（100秒）

**【スライド内容】**
- 見出し: 主要項目① 利用範囲の定義
- 表:
  | 分類 | 例 |
  |------|-----|
  | 利用推奨 | アイデア出し、文章校正、議事録要約 |
  | 条件付き許可 | 顧客向け資料の下書き（要上長確認） |
  | 利用禁止 | 機密情報の入力、最終成果物としての無確認利用 |
- キーメッセージ: 「何にAIを使ってよいか」を明確にする

**【ナレーション】**
ここからはガイドラインに含めるべき主要項目を4つ見ていきます。まず1つ目は利用範囲の定義です。これは「何にAIを使ってよくて、何に使ってはいけないか」を明確にするものです。たとえば、社内向けのアイデア出しや文章校正、議事録の要約は利用推奨とする。顧客向け資料の下書きは使ってもよいが、必ず上長の確認を経ること。一方で、機密情報をAIに入力することや、AIの出力を人間が確認せずにそのまま最終成果物として使うことは禁止する。このように3段階で分類するとわかりやすくなります。大切なのは、曖昧な表現を避けること。「適切に使いましょう」ではなく、具体的にどの業務にどのレベルで使えるかを示すことで、現場が迷わずに済みます。

## スライド 6: 主要項目② 禁止事項の明確化（100秒）

**【スライド内容】**
- 見出し: 主要項目② 禁止事項の明確化
- リスト（禁止事項の例）:
  1. 個人情報・顧客情報のAIサービスへの入力
  2. AI生成コンテンツの無断での社外公開
  3. 社内承認なしでの有料AIサービスの契約
  4. AIを使った他者の著作物の複製・改変
  5. 人事評価・採用判断をAIに委ねること
- 注意書き: 「やってはいけないこと」を具体的に列挙する

**【ナレーション】**
2つ目の主要項目は禁止事項の明確化です。ガイドラインで最も重要なパートと言っても過言ではありません。具体的にどんな行為を禁止するか、例を見てみましょう。まず、個人情報や顧客情報をAIサービスに入力すること。これは情報漏えいに直結します。次に、AIが生成したコンテンツを社外に公開する場合は必ず承認を得ること。無断公開は著作権トラブルのもとです。また、個人判断で有料AIサービスを契約することも禁止すべきです。シャドーITとなり、セキュリティ管理ができなくなります。さらに、AIを使って他者の著作物を複製・改変することや、人事評価や採用判断をAIだけに委ねることも禁止事項に含めるべきです。ポイントは、抽象的に「不適切な利用は禁止」とするのではなく、具体的な行為を列挙すること。社員が「これはやっていいのかな」と迷った時に、明確に判断できるようにすることが大切です。

## スライド 7: 主要項目③ データ取扱いルール（100秒）

**【スライド内容】**
- 見出し: 主要項目③ データ取扱いルール
- フロー図:
  - データを分類 → 公開情報 / 社内限定 / 機密 / 個人情報
  - 公開情報 → AI利用OK
  - 社内限定 → 社内AIのみOK（外部サービスNG）
  - 機密・個人情報 → AI利用禁止
- 補足: 「オプトアウト設定」の確認を義務化

**【ナレーション】**
3つ目はデータ取扱いルールです。AIに入力するデータをどう管理するかは、ガイドラインの核心部分です。まず、社内のデータを4段階に分類します。公開情報、社内限定情報、機密情報、個人情報の4つです。公開情報であれば外部AIサービスへの入力もOK。社内限定情報は、自社で管理するAI環境でのみ利用可能。機密情報と個人情報はAIへの入力を一切禁止。このようにデータの分類とAI利用の可否を紐づけることで、現場が判断しやすくなります。もう1つ重要なのが、外部AIサービスのオプトアウト設定です。多くのAIサービスは、入力データを学習に使わない設定を提供しています。この設定を必ず有効にすることをルール化しましょう。データの分類基準が曖昧だと現場は判断に迷います。各部門のデータ資産を棚卸しして、具体的に何が機密で何が公開情報なのかを整理しておくことが前提になります。

## スライド 8: 主要項目④ 承認フロー（100秒）

**【スライド内容】**
- 見出し: 主要項目④ 承認フロー
- フロー図:
  1. 利用申請（AIで何をしたいか）
  2. 上長承認（利用範囲内か確認）
  3. 情報セキュリティ部門チェック（データ分類確認）
  4. 利用開始
  5. 定期レビュー（月次で利用状況を確認）
- 補足: 新しいAIツールの導入時は情シスの事前承認が必要

**【ナレーション】**
4つ目の主要項目は承認フローです。AIを使い始める前に、誰がどのように承認するかの流れを決めておきます。基本的なフローはこうです。まず社員が利用申請を出します。「この業務にこのAIツールを使いたい」という申請です。次に上長が利用範囲内かを確認して承認します。データの機密度が高い場合は情報セキュリティ部門のチェックも入れます。承認を経て利用開始。そして月次で利用状況をレビューします。ここで大事なのは、フローを重くしすぎないことです。日常的なAI利用、たとえば文章校正やアイデア出しにまで毎回申請が必要だと、誰も使わなくなります。利用推奨の範囲は包括的に承認し、条件付き許可の範囲だけ個別に承認する。この段階的な設計が使われるガイドラインのコツです。また、新しいAIツールを導入する際は、情報システム部門の事前承認を必須とし、シャドーITを防ぎましょう。

## スライド 9: 他社事例① 大手IT企業の例（100秒）

**【スライド内容】**
- 見出し: 他社事例① 大手IT企業のアプローチ
- 事例:
  - 全社員にAIリテラシー研修を義務化
  - 社内専用AIプラットフォームを構築（外部流出リスクを排除）
  - AI利用ログを記録し、四半期ごとに監査
  - 「AI倫理委員会」を設置し、利用基準を継続的にアップデート
- ポイント: 「ルールを作って終わり」ではなく、運用体制を整備

**【ナレーション】**
ここからは他社の事例を見ていきましょう。まず大手IT企業の例です。この企業では4つの施策を同時に進めました。第一に、全社員にAIリテラシー研修を義務化。ガイドラインを配るだけでなく、なぜそのルールがあるのかを研修で丁寧に説明しています。第二に、社内専用のAIプラットフォームを構築しました。外部のAIサービスを使わせるのではなく、自社環境内でAIを利用できるようにしたのです。これにより、データの外部流出リスクを根本的に排除しています。第三に、AI利用ログを全て記録し、四半期ごとに監査を実施。不適切な利用がないかをチェックしています。第四に、AI倫理委員会を設置し、技術の進化に合わせてガイドラインを継続的にアップデートしています。この事例のポイントは、「ルールを作って終わり」ではなく、研修、インフラ、モニタリング、ガバナンスという運用体制を一体で整備していることです。

## スライド 10: 他社事例② 中小企業の段階的導入（100秒）

**【スライド内容】**
- 見出し: 他社事例② 中小企業の段階的アプローチ
- ステップ図:
  - Phase 1（1ヶ月目）: 最低限の禁止事項だけ定めてAI利用を開始
  - Phase 2（3ヶ月目）: 利用実態を調査し、利用範囲とデータルールを策定
  - Phase 3（6ヶ月目）: 承認フローと研修プログラムを整備
- ポイント: 完璧を目指さず、まず始めて段階的に改善する

**【ナレーション】**
次に中小企業の事例です。大手企業のように最初から完璧な体制を作るのは難しい。そこでこの企業は段階的なアプローチを取りました。Phase 1では最低限の禁止事項、つまり「個人情報と機密情報はAIに入れない」「AI出力はそのまま社外に出さない」。この2つだけ決めて、まずAI利用を開始しました。Phase 2として3ヶ月後に社員のAI利用実態を調査。どんなツールをどんな業務に使っているかを把握した上で、利用範囲の分類とデータ取扱いルールを策定しました。Phase 3で6ヶ月後に承認フローと研修プログラムを整備。この時点で正式なガイドラインとして社内に公開しました。この事例のポイントは「完璧を目指さず、まず始める」ことです。最初から100点のガイドラインを作ろうとすると、いつまでも完成しません。まず最低限のルールで始めて、運用しながら改善していく。特にリソースが限られる中小企業には、この段階的アプローチが効果的です。

## スライド 11: 自社で始める5つのステップ（100秒）

**【スライド内容】**
- 見出し: 自社で始める5つのステップ
- 5つのカード:
  1. 現状把握 — 社員のAI利用状況を調査する
  2. リスク評価 — 自社にとっての最大リスクを特定する
  3. ドラフト作成 — 4つの主要項目を骨子にガイドラインを起草
  4. パイロット運用 — 一部門で試行してフィードバックを収集
  5. 全社展開 — 研修と合わせてガイドラインを正式リリース
- 補足: 定期的な見直し（少なくとも年2回）を予定に組み込む

**【ナレーション】**
では最後に、皆さんの会社でガイドラインを作る際の5つのステップを整理します。ステップ1は現状把握。まず社員がどんなAIツールをどのように使っているかを調査します。アンケートやヒアリングで実態を把握しましょう。ステップ2はリスク評価。自社にとって最も大きなリスクは何かを特定します。扱うデータの種類や業界の規制によって優先すべきリスクは変わります。ステップ3はドラフト作成。今日学んだ4つの主要項目、利用範囲、禁止事項、データ取扱い、承認フローを骨子にガイドラインを起草します。ステップ4はパイロット運用。いきなり全社展開するのではなく、まず一部門で試行します。現場のフィードバックを反映して使いやすいガイドラインに仕上げましょう。ステップ5は全社展開。研修と合わせてガイドラインを正式にリリースします。そして忘れてはいけないのが定期的な見直しです。AI技術は急速に進化するので、少なくとも年2回はガイドラインを更新する予定を組み込んでおきましょう。
