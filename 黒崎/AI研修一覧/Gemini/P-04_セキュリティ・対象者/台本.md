## スライド 4: CMEK・EKM・VPC Service Controls とは

**画面:**
```
CMEK・EKM・VPC Service Controls の3段階

1️⃣ CMEK（Customer Managed Encryption Keys）
   ├─ データを暗号化する『鍵』を、顧客が管理
   ├─ Google Cloud Key Management Service で一括管理
   └─ 利点：鍵を自分たちで回転・破棄できる

2️⃣ EKM（External Key Manager）
   ├─ 暗号化の『鍵』を社内システムに置く
   ├─ HSM（ハードウェアセキュリティモジュール）と連携
   └─ 利点：Google にも鍵を見られない（最高レベルの秘匿性）

3️⃣ VPC Service Controls
   ├─ Gemini へのアクセスを『仮想要塞』の中に制限
   ├─ 社内ネットワーク経由でのみアクセス可能
   └─ 利点：データが社内ネットワークを出ない
```

**ナレーション:**
「ここから3つのセキュリティ機能を説明します。ただし、難しい用語が出てくるので、一度では理解できなくても大丈夫です。

『CMEK』をご説明します。通常、クラウドサービスのデータは『Googleが保有する鍵』で暗号化されます。CMEKを使うと『顧客であるあなたの会社が暗号化の鍵を持つ』ことになります。メリットは、鍵を定期的に交換したり、必要に応じて破棄したりできることです。金融機関や大企業の多くがこれを導入しています。

『EKM』はさらに高いレベルです。暗号化の『鍵』を、Google Cloud ではなく『自社のオンプレミスシステムに置く』のです。これにより『Google にさえデータの内容を見られない』状態を作ります。最高レベルのセキュリティが必要な企業向けです。

『VPC Service Controls』は、ネットワークレベルの制御です。Gemini へのアクセスを『仮想的な壁の中に閉じ込める』ので、社内ネットワークの外からはアクセスできなくなります。」

**音声時間: 約70秒**

## スライド 5: 暗号化キー管理：3つのレベル

**画面:**
```
セキュリティレベル別 暗号化キー管理

🟩 標準（Google管理）
   ├─ デフォルト
   ├─ Google が鍵を持つ
   └─ ほとんどの企業が選択

🟨 中級（CMEK）
   ├─ 金融・医療・大企業
   ├─ 顧客が鍵を管理
   └─ 鍵交換・破棄が自由

🟥 高度（EKM/HSM）
   ├─ 最高機密レベル
   ├─ 社内システムに鍵を置く
   └─ Google さえ中身を見られない
```

**ナレーション:**
「セキュリティとコストのバランスから、3段階を用意しています。

『標準』は、Google に全て任せる方式です。ほぼすべての機能が使えて、コストも安いです。中小企業・スタートアップはこれで十分です。

『中級』の CMEK は、『暗号化の鍵を自社で持ちたい』という企業向けです。毎年鍵を新しくしたり、辞めるときに全データを破棄したり、そういった『鍵に対するコントロール』が手に入ります。

『高度』の EKM・HSM は、『Google にさえデータを見られたくない』という最高レベルのセキュリティが必要な企業向けです。ただし、初期設定が複雑で、導入に数ヶ月かかることもあります。

皆さんの企業がどのレベルを必要としているかは、『業界規制』『取り扱うデータの機密度』で判断します。」

**音声時間: 約60秒**

## スライド 6: コンプライアンス認証ガイド

**画面:**
```
Gemini Enterprise が対応するコンプライアンス認証

🏥 HIPAA（医療業界）
   └─ 患者情報の取り扱いに必須

🏛️ FedRAMP（米国政府）
   └─ 米国政府機関との取引に必須

🌍 ISO 27001・27018（国際標準）
   └─ 大企業・グローバル企業のデファクト

📊 SOC 2 Type II（監査報告）
   └─ セキュリティ監査で要求される

💳 PCI DSS（クレジットカード業界）
   └─ 決済システムに必須
```

**ナレーション:**
「次に『コンプライアンス認証』について説明します。これは『Gemini Enterprise が『この基準を満たしていますよ』と証明するもの』です。

『HIPAA』は医療業界の規制です。患者の医療記録を扱う企業であれば、このGemini での処理が HIPAA に準拠していることが必須です。

『FedRAMP』は米国政府の基準です。アメリカの政府機関と取引する企業、例えば防衛関連企業なら、FedRAMP認証が必須です。

『ISO 27001・27018』は、国連や ISO が定めた『情報セキュリティの国際標準』です。日本の大企業やグローバル企業のほぼすべてが取得しており、ビジネスパートナーから要求されることもあります。

『SOC 2 Type II』は『監査報告書』です。Gemini Enterprise のセキュリティが、外部監査機関により『安全です』と認めていることを示します。

『PCI DSS』はクレジットカード業界の基準です。決済情報を扱う企業ならば、絶対に必要です。」

**音声時間: 約75秒**
