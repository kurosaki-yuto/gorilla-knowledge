## スライド 3: 100万トークンとは（75秒）

**【スライド内容】**
- 見出し: 100万トークンのスケール感
- 換算表:
  - 新書1冊: 約12万トークン
  - 100万トークン = 新書約8冊分
  - A4文書1ページ: 約500〜700トークン
  - 100万トークン = A4約1,400〜2,000ページ
- 比較:
  - Claude: 100万トークン（Proプラン $20/月）
  - ChatGPT Plus: 256Kトークン（$20/月）
  - ChatGPT Pro: 400Kトークン（$200/月）

**【ナレーション】**
まず100万トークンの規模感を整理しましょう。日本語では、新書1冊がおよそ12万トークンです。つまり100万トークンは新書にして約8冊分。分厚いビジネス書でも4〜5冊は丸ごと入ります。A4のビジネス文書に換算すると、1ページが約500〜700トークンなので、1,400〜2,000ページ分です。会社の年次報告書が100ページ前後ですから、10社分以上を同時に読み込めます。ChatGPTとの比較で見ると、ChatGPT Plusは256Kトークンで、Claudeの約4分の1。ChatGPT Proでも400Kトークンで、Claudeの半分以下です。しかもChatGPT Proは月200ドル、Claude Proは月20ドル。10分の1の価格で2.5倍のコンテキストを使えるわけです。長文処理においてClaudeのコストパフォーマンスは圧倒的です。

## スライド 4: 長文処理の実演1 — PDF一括分析（75秒）

**【スライド内容】**
- 見出し: 実演1：100ページPDFの一括分析
- Before: 手作業で4〜8時間（読む→メモ→整理→要約）
- After: Claude約30秒（アップロード→プロンプト→完成）
- プロンプト例:
  ```
  添付のPDFを分析し、以下の形式で出力してください。
  1. エグゼクティブサマリー（200字以内）
  2. 主要データポイントTOP10（表形式）
  3. 重要な示唆TOP5（優先度順に箇条書き）
  4. 注意すべきリスク要因（あれば）
  ```

**【ナレーション】**
では実演に入ります。まず1つ目、100ページのPDF一括分析です。たとえば業界レポートや社内の年次報告書を想像してください。手作業なら、まず全ページを読み、重要箇所にマーカーを引き、メモを整理して、最後にサマリーを書く。少なくとも4〜8時間かかります。Claudeならこうです。PDFをアップロードし、画面に表示しているプロンプトを入力。「エグゼクティブサマリーを200字以内で」「主要データポイントTOP10を表形式で」「重要な示唆TOP5を優先度順に」「リスク要因があれば」。約30秒で全てが返ってきます。ポイントは、出力フォーマットを明確に指定していること。「分析してください」だけだと出力が散漫になりますが、形式を指定すると、すぐに使える構造化された結果が得られます。この差は非常に大きい。

## スライド 5: 長文処理の実演2 — 複数文書の比較分析（75秒）

**【スライド内容】**
- 見出し: 実演2：競合3社の決算資料を同時比較
- 手順:
  1. 3社の決算資料PDF（各30〜50ページ）をアップロード
  2. 比較観点を指定してプロンプトを入力
- プロンプト例:
  ```
  添付の3社の決算資料を比較分析してください。
  
  比較軸:
  - 売上高・営業利益・利益率の3期比較
  - セグメント別の成長率
  - 設備投資・R&D投資の方針
  - 各社が挙げているリスク要因
  
  出力形式: 比較表 + 総合所見（各社の強み・弱み）
  ```

**【ナレーション】**
2つ目は、複数文書の比較分析です。たとえば競合3社の決算資料を同時に分析する場面。各社30〜50ページの決算資料を3社分、合計100〜150ページ。これを一括でClaudeに読み込ませます。100万トークンなら余裕です。プロンプトでは比較軸を明示的に指定します。「売上高・営業利益・利益率の3期比較」「セグメント別の成長率」「設備投資とR&D投資の方針」「各社が挙げているリスク要因」。出力形式も「比較表＋総合所見」と指定。すると、3社を横並びにした比較表と、各社の強み・弱みの総合分析が返ってきます。コンサルファームに依頼すれば数十万円、自分でやれば丸1日かかる作業です。それが数分で完了する。これが100万トークンの真の価値です。

## スライド 6: 長文処理の実演3 — 議事録の構造化（75秒）

**【スライド内容】**
- 見出し: 実演3：長い議事録の構造化
- よくある課題: 2時間の会議 → 議事録30ページ → 誰も読まない
- プロンプト例:
  ```
  添付の議事録を以下の形式に構造化してください。
  
  ## 決定事項（決定内容・担当者・期日）
  ## アクションアイテム（内容・担当者・期日・優先度）
  ## 未解決課題（内容・次回議論予定・関係者）
  ## 議論のポイント（賛成/反対の論点整理）
  
  ※ 曖昧な表現は原文を引用し、解釈を加えないこと
  ```

**【ナレーション】**
3つ目は、長い議事録の構造化です。多くの企業で「2時間の会議の議事録が30ページあるけど、結局何が決まったか分からない」という問題が起きています。Claudeなら、議事録の全文を読み込ませて、決定事項・アクションアイテム・未解決課題・議論のポイントに自動分類できます。特に便利なのがアクションアイテムの抽出。担当者と期日を自動で拾い上げてくれます。プロンプトの最後に「曖昧な表現は原文を引用し、解釈を加えないこと」と指示しているのがポイントです。AIが勝手に解釈して「こう決まった」と断言するのは危険です。「Aの方向で進める雰囲気だった」程度の発言は、そのまま原文を引用させる。この一行を加えるだけで、出力の信頼性が大きく変わります。

## スライド 7: 効果的な長文分析のコツ（75秒）

**【スライド内容】**
- 見出し: 長文分析を成功させる3つのコツ
- 1. 分析の観点を明確に指定する
  - NG: 「この資料を分析して」
  - OK: 「コスト構造・競合優位性・成長リスクの3点で分析して」
- 2. 出力フォーマットを指定する
  - 表、箇条書き、番号付きリスト、Markdown見出し
- 3. 段階的に深掘りする
  - まず全体要約 → 気になる箇所を詳細分析

**【ナレーション】**
長文分析で安定した結果を得るためのコツを3つ紹介します。1つ目は、分析の観点を明確に指定すること。「この資料を分析してください」ではClaudeは何をすればいいか迷います。「コスト構造・競合優位性・成長リスクの3点で分析してください」と指定すれば、焦点が絞られた深い分析が返ってきます。2つ目は、出力フォーマットの指定。表で欲しいのか、箇条書きでいいのか、Markdownの見出し付きで構造化したいのか。フォーマットを指定するだけで、後の加工作業が劇的に減ります。3つ目は、段階的な深掘り。最初から「全てを詳細に分析して」と頼むと、情報が多すぎて使いにくくなります。まず全体要約を出させ、気になるポイントを見つけてから「この部分をもっと詳しく」と深掘りする。この2段階アプローチが最も効率的です。

## スライド 8: 長文処理の落とし穴（75秒）

**【スライド内容】**
- 見出し: 長文処理で気をつけるべきこと
- 1. 「Needle in a Haystack」問題
  - 大量テキストの中の小さな情報を見落とす可能性
  - 対策: 「特に○○に注目して」と焦点を絞る
- 2. 文書の途中が薄くなる傾向
  - 先頭と末尾は精度が高いが、中盤の情報は弱くなる場合がある
  - 対策: 重要情報の位置が偏っている場合は明示する
- 3. ハルシネーション（事実の捏造）
  - 長文ほど「それっぽい」が事実でない内容が紛れ込むリスク
  - 対策: 「根拠を原文から引用して」と指示する

**【ナレーション】**
長文処理には落とし穴もあります。正直にお伝えしておきます。1つ目は「Needle in a Haystack」問題。大量のテキストの中に埋もれた小さな情報を見落とすことがあります。対策は、「特にこの点に注目して」と焦点を明示すること。2つ目は、文書の中盤が薄くなる傾向。先頭と末尾の情報は正確に拾いますが、中盤の情報は見落とされやすい場合があります。重要な情報が中盤に集中している場合は、「ページ50〜70に重要なデータがあるので特に注意してください」と伝えてください。3つ目はハルシネーション。長い文書ほど「それっぽい」が事実でない要約が紛れ込むリスクがあります。対策は「根拠となる原文を引用してください」と指示すること。引用を求めると、AIは存在しない情報を作りにくくなります。

## スライド 9: Web検索機能の概要（75秒）

**【スライド内容】**
- 見出し: Claude Web検索 — リアルタイム情報へのアクセス

  - 引用付き回答（ソースURLが表示される）
  - Claudeが検索の必要性を自動判断
- 設定: claude.ai のチャット画面でWeb検索トグルをON
- 対応プラン: Pro以上（無料プランでは制限あり）

**【ナレーション】**
ここからはWeb検索機能の話に移ります。以前のClaudeは学習データの期限までの情報しか持っていませんでしたが、Web検索機能により最新情報にアクセスできるようになりました。使い方は簡単。claude.aiのチャット画面でWeb検索のトグルをONにするだけです。あとは普通に質問すれば、Claudeが検索の必要性を自動判断し、必要に応じてWebを検索し、結果を統合して回答します。重要なのは、引用付きで回答が返ってくること。「〇〇によると」という記述にソースURLが紐づいているので、情報の出典を確認できます。これにより、AIの回答を鵜呑みにせず、元ソースで裏取りできます。Pro以上のプランで利用可能で、無料プランでは検索回数に制限があります。

## スライド 10: ChatGPTの検索 vs Claudeの検索（75秒）

**【スライド内容】**
- 見出し: 検索機能の質的な違い
- ChatGPTの検索:
  - 検索結果をまとめて分かりやすく要約
  - 幅広いソースから情報を収集
  - 「何が書いてあるか」を伝える
- Claudeの検索:
  - 検索結果を分析的に統合
  - 情報の矛盾や差異を指摘
  - 「何が読み取れるか」を分析する
- 例: 「2026年の半導体市場予測」で比較

**【ナレーション】**
ChatGPTとClaudeのWeb検索は、同じ「検索」でも性質が異なります。ChatGPTの検索は「情報のまとめ」が得意。複数のソースから情報を集め、分かりやすく要約してくれます。読みやすく、概要を掴むには最適です。一方、Claudeの検索は「情報の分析」寄り。複数ソースの情報を突き合わせ、矛盾点や差異を指摘する傾向があります。たとえば「2026年の半導体市場予測」と聞くと、ChatGPTは主要リサーチ機関の予測をまとめて提示します。Claudeは「A社は成長率15%と予測する一方、B社は8%としており、この差異はAI半導体の需要見通しの違いに起因すると考えられる」という分析を返すことがあります。どちらが良いという話ではなく、速報的に情報を掴みたいならChatGPT、情報の意味を深く理解したいならClaudeという使い分けです。
