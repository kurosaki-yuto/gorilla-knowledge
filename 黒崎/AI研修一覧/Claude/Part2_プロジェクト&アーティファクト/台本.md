## スライド 3: プロジェクト機能とは（75秒）

**【スライド内容】**
- 見出し: プロジェクト = 業務専用AIアシスタント
- 概念図:
  - 通常のチャット: 毎回ゼロから説明が必要
  - プロジェクト: 業務知識を事前に読み込み済み
- 2つの構成要素:
  1. ナレッジベース（業務資料のアップロード）
  2. カスタム指示（AIへの事前指示）
- 利用条件: Proプラン以上（月$20）

**【ナレーション】**
プロジェクト機能を一言で言うと、「業務専用のAIアシスタントを作る機能」です。通常のClaudeチャットでは、毎回「私は営業部で、こういう製品を扱っていて、ターゲットはこういう顧客で」と説明する必要があります。しかしプロジェクトを使えば、製品カタログ、営業マニュアル、過去の提案書などをナレッジベースとして事前にアップロードしておけます。さらにカスタム指示で「あなたは当社の営業アシスタントです。提案書は必ず当社のフォーマットに従ってください」と役割を設定できます。これにより、プロジェクト内でチャットを始めると、Claudeは最初からあなたの業務を理解した状態で応答します。ChatGPTの「GPTs」に似た機能ですが、プロジェクトは100万トークン分の資料をアップロードできる点が大きな違いです。

## スライド 4: ナレッジベースの仕組み（75秒）

**【スライド内容】**
- 見出し: ナレッジベース — 業務資料をAIに読み込ませる
- アップロード可能なファイル:
  - PDF、Word、Excel、PowerPoint
  - テキスト、CSV、JSON
  - コードファイル（.py, .js, .ts等）
- 容量: プロジェクトあたり最大約200MB
- 仕組み: ファイル内容がコンテキストに自動挿入される
- ポイント: 必要な情報だけを厳選してアップロード

**【ナレーション】**
ナレッジベースは、プロジェクトに業務資料をアップロードする機能です。PDF、Word、Excel、PowerPointなど主要なビジネス文書に対応しています。コードファイルもアップロード可能です。アップロードした資料は、そのプロジェクト内の全チャットでClaudeが参照できるようになります。ここで重要なのは「何でもかんでもアップロードしない」こと。100万トークンは大容量ですが、無関係な情報が多いとClaudeの回答精度が下がります。例えば営業プロジェクトなら、製品カタログ、価格表、過去の成功提案書、よくある質問集など、営業活動に直結する資料に絞りましょう。社内報や全社会議の議事録まで入れる必要はありません。「このAIに何を知っていてほしいか」を考えて、厳選することがコツです。

## スライド 5: カスタム指示の書き方（75秒）

**【スライド内容】**
- 見出し: カスタム指示 — AIの役割・口調・ルールを定義
- テンプレート:
  ```
  あなたは[会社名]の[部署]の業務アシスタントです。
  
  【役割】
  - [具体的な業務内容]を支援する
  
  【ルール】
  - 社内用語は[用語集]に従う
  - 出力は必ず[フォーマット]で作成する
  - 不明点は推測せず確認する
  
  【トーン】
  - ビジネス敬語で簡潔に回答する
  ```
- ポイント: 曖昧さを排除し、具体的に書く

**【ナレーション】**
カスタム指示は、プロジェクト内でClaudeがどう振る舞うかを定義するルールです。画面に表示しているテンプレートが基本形です。まず「役割」。Claudeに何者として振る舞うかを指定します。「あなたは当社の法務アシスタントです」のように具体的に書きます。次に「ルール」。出力形式、使用する用語、禁止事項などを箇条書きで並べます。例えば「金額は税込みで表示」「競合他社名は出さない」「不確かな法的判断は必ず弁護士確認を推奨する」など。最後に「トーン」。敬語か、カジュアルか、技術的か。社外向け文書を作るプロジェクトなら「ビジネス敬語で簡潔に」、社内ブレスト用なら「フランクに、箇条書きで」と設定します。この指示が具体的であるほど、毎回のチャットで一貫した品質の回答が返ってきます。

## スライド 6: プロジェクト作成手順（75秒）

**【スライド内容】**
- 見出し: プロジェクト作成 — 4ステップ
- 手順:
  1. プロジェクト名を入力（例:「営業提案書アシスタント」）
  2. カスタム指示を設定（役割・ルール・トーン）
  3. ナレッジベースにファイルをアップロード
  4. テストチャットで動作確認 → 指示を調整
- 画面イメージ: Claude左サイドバーの「Projects」セクション

**【ナレーション】**
プロジェクトの作成手順を見ていきましょう。まず、Claudeの画面左側にある「Projects」からプロジェクトを新規作成します。ステップ1で、わかりやすいプロジェクト名を付けます。「営業」ではなく「営業提案書アシスタント」のように、目的が一目でわかる名前にしましょう。ステップ2で、カスタム指示を書きます。先ほどのテンプレートを使い、役割・ルール・トーンを定義します。ステップ3で、ナレッジベースにファイルをアップロードします。ドラッグ&ドロップで複数ファイルを一括追加できます。ステップ4が最も重要です。テストチャットを開いて、実際の業務で使うような質問を投げてみてください。「この製品の価格を教えて」「提案書のドラフトを作って」と聞いて、期待通りの回答が返るか確認します。不十分なら指示を調整し、繰り返します。

## スライド 9: アーティファクトとは（75秒）

**【スライド内容】**
- 見出し: アーティファクト = リアルタイム成果物生成
- 概念図:
  - 通常のチャット: テキストで回答
  - アーティファクト: 画面右側にプレビュー付き成果物が出現
- 特徴:
  - 会話とは別のパネルに独立表示
  - その場で編集・修正・バージョン管理が可能
  - コピー&ダウンロードですぐ使える
- 自動判断: 成果物として適切な回答はClaudeが自動でアーティファクト化

**【ナレーション】**
ここからはアーティファクトの解説に入ります。アーティファクトとは、Claudeが会話の中でリアルタイムに成果物を生成し、プレビュー表示する機能です。通常のAIチャットでは、コードやHTMLを生成しても結果はテキストとして表示されるだけです。しかしClaudeのアーティファクトでは、生成されたHTMLがレンダリングされた状態で画面右側に表示されます。Mermaidのフロー図はそのまま図として見えますし、Reactコンポーネントは実際に動作するインタラクティブな状態で表示されます。成果物に修正を加えたい場合は「表の列を追加して」「色を青系に変えて」とチャットで指示するだけ。バージョン履歴も残るので、前のバージョンに戻すことも可能です。コピーやダウンロードもワンクリック。まさに「AIとの共同制作環境」です。

## スライド 10: アーティファクト対応形式（75秒）

**【スライド内容】**
- 見出し: アーティファクトで作れるもの
- 対応形式一覧:
  | 形式 | 用途例 |
  | テキスト/Markdown | レポート、議事録、マニュアル |
  | HTML | Webページ、分析レポート、ランディングページ |
  | SVG | ロゴ、図解、インフォグラフィック |
  | Mermaid | フローチャート、組織図、シーケンス図 |
  | React (JSX) | ダッシュボード、計算ツール、インタラクティブUI |
  | コード | Python、JavaScript、SQL等のスクリプト |

**【ナレーション】**
アーティファクトで作れるものを整理しましょう。まずテキストとMarkdown。レポートや議事録の構造化されたドキュメントです。次にHTML。表やチャートを含む分析レポート、簡単なWebページが作れます。CSSも含められるので、見た目の整った成果物になります。SVGはベクター画像。図解やインフォグラフィック、簡単なロゴデザインに使えます。Mermaidは図表記述言語で、フローチャートや組織図、システムのシーケンス図をテキストから自動生成します。そしてReact。これが最も強力で、Rechartsなどのライブラリを使ったインタラクティブなダッシュボードや計算ツールを、その場で動作する状態で生成できます。最後にコード。PythonやJavaScriptのスクリプトをシンタックスハイライト付きで表示します。この後、実際に3つの形式で成果物を作ってみましょう。

## スライド 11: 実演1 — 競合分析HTMLレポート（75秒）

**【スライド内容】**
- 見出し: 実演1: HTMLで競合分析レポートを生成
- プロンプト例:
  ```
  以下の3社の競合分析HTMLレポートをアーティファクトで作成してください。
  - A社: クラウド会計ソフト、月額2,980円、中小企業向け
  - B社: クラウド会計ソフト、月額1,980円、個人事業主向け

  ```

**【ナレーション】**

## スライド 12: 実演2 — 業務フローのMermaid図（75秒）

**【スライド内容】**
- 見出し: 実演2: Mermaidで業務フローを可視化
- プロンプト例:
  ```
  以下の受注処理フローをMermaid図でアーティファクトとして作成してください。
  ```

**【ナレーション】**

## スライド 13: 実演3 — KPIダッシュボード（75秒）

## スライド 14: ChatGPT Canvas vs Claude Artifacts（75秒）

**【スライド内容】**
- 見出し: Canvas vs Artifacts — 何が違う？
- 比較表:
  | 比較軸 | ChatGPT Canvas | Claude Artifacts |
  | メイン用途 | テキスト・コードの協調編集 | 多形式の成果物生成 & プレビュー |
  | テキスト編集 | 直接インライン編集が可能 | チャットで修正指示 |
  | HTMLプレビュー | 非対応（コードのみ） | リアルタイムレンダリング |
  | Reactプレビュー | 非対応 | リアルタイム実行 |
  | Mermaid | 非対応 | リアルタイムレンダリング |
  | SVG | 非対応 | リアルタイムレンダリング |
  | バージョン管理 | あり | あり |
  | コード実行 | Python実行可能 | プレビューのみ（実行環境なし） |

**【ナレーション】**
ChatGPTの「Canvas」とClaudeの「Artifacts」は似た機能に見えますが、設計思想が異なります。Canvasはテキストとコードの「協調編集」に特化しています。生成された文章やコードを直接クリックして部分的に修正でき、Googleドキュメントに近い操作感です。一方Artifactsは「多形式の成果物生成とプレビュー」に強みがあります。HTMLをレンダリングした状態で表示し、Reactコンポーネントをその場で実行し、Mermaid図を描画する。Canvasにはこのプレビュー機能がありません。ただしCanvasにはPythonコードを実行できるという強みがあります。データ分析でグラフを描く場合はCanvasが有利です。まとめると、文書の共同編集やPython実行にはCanvas、HTML・React・図表のビジュアル成果物にはArtifacts、という使い分けになります。
