## スライド 3: Claude Codeとは（75秒）

**【スライド内容】**
- 見出し: Claude Code — ターミナルで動くAIコーディングエージェント
- 概要:
  - Anthropic公式のCLI（コマンドラインインターフェース）ツール
  - ターミナルで `claude` と打つだけで起動
  - AIがファイルを直接読み書き・実行する「エージェント型」
- 利用条件: Claude Pro プラン以上（月20ドル〜）
- インストール: `npm install -g @anthropic-ai/claude-code`

**【ナレーション】**
Claude Codeとは、Anthropicが公式に提供しているCLIツールです。CLIとは「コマンドラインインターフェース」の略で、ターミナル（黒い画面）から操作するソフトウェアのことです。普通のClaude.aiはブラウザで会話しますが、Claude Codeはターミナルで動きます。インストールはnpmというパッケージマネージャーで一行のコマンドを実行するだけ。あとはターミナルで「claude」と入力すれば起動します。最大の特徴は「エージェント型」であること。チャットでコードの書き方を相談するのではなく、AIがあなたのPC上のファイルを直接読み書きします。「この関数にバグがあるから直して」と言えば、AIがソースコードを開き、該当箇所を見つけ、修正し、保存してくれます。利用にはClaude Proプラン以上が必要で、月20ドルから使えます。

## スライド 5: CLAUDE.md — プロジェクトの「取扱説明書」（75秒）

**【スライド内容】**
- 見出し: CLAUDE.md — AIにプロジェクトのルールを教える
- 仕組み:
  - プロジェクトのルートディレクトリに `CLAUDE.md` を配置
  - Claude Codeが起動時に自動で読み込む
  - プロジェクトの構成・コーディング規約・デプロイ方法などを記述
- 記述例:
  - 「TypeScriptで記述。テストはVitest」
  - 「コミットメッセージは日本語で」
  - 「srcフォルダ以外は変更しない」
- 階層構造: サブディレクトリごとに追加の CLAUDE.md を配置可能

**【ナレーション】**
CLAUDE.mdは、Claude Codeにとっての「プロジェクトの取扱説明書」です。プロジェクトフォルダの一番上にCLAUDE.mdというファイルを置いておくと、Claude Codeは起動時にこれを自動で読み込みます。ここに「このプロジェクトはTypeScriptで書かれている」「テストはVitestを使う」「コミットメッセージは日本語で」などのルールを書いておけば、AIはそのルールに従って作業します。これが非常に重要なのは、AIが「勝手な判断」をしないようコントロールできるからです。たとえば「srcフォルダ以外は変更しない」と書けば、設定ファイルを勝手にいじることはありません。サブディレクトリごとに追加のCLAUDE.mdを配置することもでき、フロントエンドとバックエンドで異なるルールを適用することも可能です。人間のチームにオンボーディング資料を渡すように、AIにもプロジェクトの作法を教えるわけです。

## スライド 9: MCPとは — AIのUSB-C（75秒）

**【スライド内容】**
- 見出し: MCP（Model Context Protocol）— AIを外部ツールに接続する共通規格
- 概要:
  - Anthropicが策定したオープン標準プロトコル
  - 「AIのUSB-C」のような存在
  - 従来: ツールごとに個別のAPI連携が必要
  - MCP: 共通の接続方式で、どのツールも同じように繋がる
- 比喩:
  - USB-C以前: 充電器もケーブルも機器ごとにバラバラ
  - USB-C以後: 1つの規格であらゆる機器に接続
  - MCPはAIとツールの接続で同じことを実現

**【ナレーション】**
ここからはMCPについて解説します。MCPとはModel Context Protocolの略で、AIを外部のツールやサービスに接続するための共通規格です。Anthropicが策定し、オープンスタンダードとして公開しています。わかりやすく言うと「AIのUSB-C」です。USB-Cが登場する前、スマートフォンの充電器はメーカーごとにバラバラでした。USB-Cが標準になったことで、1本のケーブルであらゆる機器を充電できるようになりました。MCPはこれと同じことをAIの世界で実現します。従来、AIからSlackを操作するにはSlack専用のAPI連携を開発し、GitHubを操作するにはGitHub専用の連携を開発する必要がありました。MCPを使えば、共通の接続方式でどのツールも同じように繋がります。しかもオープン標準なので、Anthropicだけでなく他のAI企業やツール開発者も採用し始めており、業界標準になりつつあります。

## スライド 10: MCPの仕組み — ホスト・クライアント・サーバー（75秒）

**【スライド内容】**
- 見出し: MCPのアーキテクチャ
- 構成図:
  - ホスト（Claude Desktop / Claude Code）
    - → MCPクライアント（接続管理）
      - → MCPサーバーA（GitHub）
      - → MCPサーバーB（Slack）
      - → MCPサーバーC（データベース）
- 3つの機能:
  1. ツール（Tools）: 外部サービスを「操作」する（メッセージ送信、PR作成）
  2. リソース（Resources）: 外部サービスから「データを読み取る」
  3. プロンプト（Prompts）: 定型の指示テンプレート

**【ナレーション】**
MCPの仕組みをもう少し詳しく見ましょう。MCPは3つの層で構成されます。まず「ホスト」。Claude DesktopやClaude Codeなど、皆さんが直接操作するアプリケーションです。次に「MCPクライアント」。ホストの内部で動き、接続を管理する仲介役です。そして「MCPサーバー」。GitHubやSlack、データベースなどの外部サービスとの橋渡しをするプログラムです。MCPサーバーは3つの機能を提供します。1つ目は「ツール」。外部サービスを操作する機能です。Slackにメッセージを送る、GitHubでPRを作成するといったアクションがこれにあたります。2つ目は「リソース」。外部サービスからデータを読み取る機能です。特定のチャンネルのメッセージを取得する、ファイルの内容を読むといった操作です。3つ目は「プロンプト」。定型の指示テンプレートで、よく使う操作をワンクリックで実行できるようにします。

## スライド 12: MCPの設定方法（75秒）

**【スライド内容】**
- 見出し: MCPサーバーの設定 — Claude Desktopの場合
- 手順:
  1. Claude Desktopの設定画面を開く
  2. 「Developer」→「Edit Config」で設定ファイルを編集
  3. MCPサーバーの情報をJSON形式で追記
- 設定例（ファイルシステム）:
  ```json
  {
    "mcpServers": {
      "filesystem": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/you/Documents"]
      }
    }
  }
  ```
- ポイント: 一度設定すれば、Claudeから自動で利用可能に

**【ナレーション】**
MCPサーバーの設定方法を見てみましょう。Claude Desktopの場合、設定画面からDeveloperメニューを開き、Edit Configボタンをクリックすると設定ファイルが開きます。ここにMCPサーバーの情報をJSON形式で追記します。たとえばファイルシステムMCPサーバーの場合、mcpServersの下にfilesystemというキーで、実行コマンドとアクセスを許可するフォルダのパスを指定します。設定を保存してClaude Desktopを再起動すると、チャット画面にMCPツールのアイコンが表示されます。あとは普通に会話するだけ。「Documentsフォルダにある先月のレポートを要約して」と聞けば、AIが自動でMCPサーバー経由でファイルを読み取り、要約してくれます。設定は最初の一回だけで、以降はClaudeが必要に応じて自動的にMCPサーバーを使います。技術的なハードルはありますが、手順通りに進めれば非エンジニアでも設定可能です。
