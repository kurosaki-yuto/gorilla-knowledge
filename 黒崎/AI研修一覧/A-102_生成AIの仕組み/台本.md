## スライド 3: 生成AIは何をしている?（80秒 / 約650文字）

**【スライド内容】**
- 見出し: 生成AIは何をしている?
- 核心: 「次の単語を予測する」を繰り返しているだけ
- フロー図: 「今日の天気は」→ 晴れ → です → 。
- たとえ: スマホの予測変換の超高性能版

**【ナレーション】**
まず、生成AIが実際に何をしているのか。答えは驚くほどシンプルです。「次に来そうな言葉を予測する」、これを何百回も繰り返しているだけなんです。たとえば「今日の天気は」と入力すると、AIは膨大なデータから学んだパターンに基づいて「次に来る言葉は"晴れ"の確率が高い」と判断します。「晴れ」を出力したら、今度は「今日の天気は晴れ」の続きを予測して「です」を出す。さらに「。」を出す。この繰り返しで、まるで人間が書いたような文章が出来上がるわけです。ここが最も大事なポイントです。AIは「考えて」文章を書いているのではなく、統計的に最もありそうな言葉を一つずつ選んでいるだけ。スマートフォンの予測変換を思い出してください。「お」と打つと「お疲れ様です」が候補に出ますよね。あの仕組みを何千億倍も高性能にしたものが、生成AIだとイメージしてください。

## スライド 4: LLMとは何か（80秒 / 約650文字）

**【スライド内容】**
- 見出し: LLM -- 大規模言語モデルの全体像
- 3カード: L=Large / L=Language / M=Model
- たとえ: 「図書館の全ての本を読んだ人」

**【ナレーション】**
ここで「LLM」という用語を押さえましょう。LLMはLarge Language Modelの略で、日本語では「大規模言語モデル」と呼ばれます。ChatGPTやClaudeの頭脳にあたる部分です。3つの単語を分解すると分かりやすくなります。まず「Large」、大規模。インターネット上の膨大なテキストを大量に読み込んでいます。その量は数兆語に及び、ニュース記事、Wikipedia、書籍、学術論文、プログラムコードなど多岐にわたります。次に「Language」、言語。主に言葉のパターンを扱う仕組みです。最近は画像や音声も扱えるマルチモーダルモデルが主流になりつつあります。最後に「Model」、モデル。学んだパターンを数学的な計算式で表現したものです。身近なたとえで言えば、世界中の図書館の本を全部読んだ人のようなもの。ただし、内容を理解しているのではなく、「こういう文脈ではこういう言葉が来やすい」というパターンを記憶している、という点が人間との大きな違いです。

## スライド 6: 学習ステップ2（75秒 / 約625文字）

**【スライド内容】**
- 見出し: 学習ステップ2: 穴埋め問題を解いて学ぶ
- 図解: 「今日は天気が＿＿ので」→「よい」を予測
- 数字: 何兆回もの反復、何千億のパラメータ調整
- たとえ: 赤ちゃんが言葉を覚える過程

**【ナレーション】**
ステップ2は「穴埋め問題を解いて学ぶ」です。具体的にはこうです。「今日は天気がよいので公園に行った」という文章から「よい」を隠して、AIに「ここに入る言葉は何か」を予測させます。最初はデタラメな回答を出しますが、正解と比較して計算式を少しずつ修正します。このプロセスを何兆回も繰り返すことで、だんだん正確に予測できるようになります。AIが持つ「パラメータ」は、いわば調整つまみのようなもの。最新モデルでは数兆個にも及びます。学習とは、この何兆個のつまみを最適な位置に合わせる作業です。赤ちゃんが言葉を覚える過程に似ています。最初は意味が分からなくても、何千回も「いただきます」を聞くうちに、食事の前にこの言葉が来ると学習する。AIも同じ原理で言葉のパターンを獲得しているのです。

## スライド 7: 学習ステップ3 RLHF（75秒 / 約625文字）

**【スライド内容】**
- 見出し: 学習ステップ3: 人間のフィードバックで磨く
- キーワード: RLHF（人間のフィードバックによる強化学習）
- 3段フロー: 回答生成 → 人間が評価 → 報酬で調整
- 効果: 安全・丁寧・正確な回答に

**【ナレーション】**
ステップ3は「人間のフィードバックで磨く」です。専門用語ではRLHF、Reinforcement Learning from Human Feedbackと呼ばれます。ステップ2だけだと、AIは文法的には正しいけれど不適切な回答をすることがあります。差別的な表現や危険な指示を平気で生成してしまう。そこで、人間の評価者がAIの回答を「良い・悪い」で採点し、良い回答のパターンを強化していきます。具体的なフローはこうです。まず、AIが同じ質問に対して複数の回答を生成する。次に、人間の評価者が回答を比較して順位をつける。最後に、その評価結果を報酬シグナルとしてAIのパラメータを調整する。このプロセスを経て、AIは単に「次の言葉を予測する」だけでなく、「人間にとって役立つ回答を生成する」方向へ磨かれていきます。ChatGPTが自然な対話ができるのは、このRLHFのおかげです。

## スライド 8: 推論の仕組み（75秒 / 約625文字）

**【スライド内容】**
- 見出し: 推論の仕組み -- 確率的に次の単語を選ぶ
- 確率分布の例: 「日本の首都は」→ 東京(92%) / 京都(3%) / 大阪(2%)
- Temperature: 低=確実、高=創造的
- 1トークンずつ生成（自己回帰型）

**【ナレーション】**
学習が終わったAIが実際に文章を生成する「推論」の仕組みを見てみましょう。「日本の首都は」と入力すると、AIは語彙の中の全ての単語に対して確率を計算します。「東京」が92パーセント、「京都」が3パーセント、「大阪」が2パーセント、といった具合です。ここで面白いのが「Temperature」というパラメータです。Temperatureが低いと、最も確率の高い単語を確実に選びます。事実を正確に答えてほしいときに有効です。逆にTemperatureを高くすると、確率が低い単語も選ばれやすくなり、創造的で意外性のある文章が生まれます。小説の執筆やブレストに向いています。そしてこのプロセスは1トークンずつ順番に行われます。1つ生成したら次の入力に加えて、また次の1トークンを予測する。これを自己回帰型生成と呼びます。ChatGPTの回答が1文字ずつ表示されるのは、まさにこの仕組みの表れです。

## スライド 9: Transformerアーキテクチャ（80秒 / 約650文字）

**【スライド内容】**
- 見出し: Transformer -- なぜ高性能なのか
- キーワード: アテンション（注意機構）
- 例: 「銀行に行った。お金を下ろす」→ 金融機関と判断
- 歴史: 2017年Google「Attention Is All You Need」

**【ナレーション】**
現在の生成AIの中核技術が「Transformer」です。2017年にGoogleの研究者が発表した「Attention Is All You Need」という論文から始まりました。この論文タイトルの通り、最大の特徴は「アテンション」、日本語で「注意機構」です。これは文章の中でどの言葉がどの言葉と関係が深いかを計算する仕組みです。たとえば「彼女は銀行に行った。お金を下ろすためだ」という文章。人間なら「銀行」が金融機関だとすぐに分かりますよね。「お金を下ろす」という文脈があるからです。もし「魚を釣るためだ」なら、川岸の意味だと判断できる。Transformer以前の技術では、文章を先頭から順番に処理するしかなく、長い文章の文脈把握が苦手でした。Transformerは文章全体を一度に見渡して、どの単語間に強い関連があるかを並列計算できます。これにより処理速度と精度が飛躍的に向上し、GPT-5シリーズ、Claude Opus 4.6、Gemini 3.1 Pro、Llama 4など全ての主要モデルがこの技術を採用しています。

## スライド 11: 限界1 ハルシネーション（80秒 / 約650文字）

**【スライド内容】**
- 見出し: 生成AIの限界1: ハルシネーション
- 定義: 事実でないことをもっともらしく生成する現象
- 実例: 存在しない論文の引用、架空の統計データ
- 原因: 「事実確認」の仕組みが存在しない

**【ナレーション】**
ここからは生成AIの限界について、仕組みの観点から解説します。最も重要な限界が「ハルシネーション」、日本語で「幻覚」と呼ばれる現象です。AIが事実でない情報をもっともらしく生成してしまうことを指します。なぜこれが起きるのか、仕組みを思い出してください。AIは「次に来そうな言葉」を確率的に選んでいるだけで、「自分が言っていることが事実かどうか」を確認する機能を持っていません。たとえば「この分野の権威であるSmith教授の2023年の研究によると」という文を生成したとき、Smith教授も論文も実在するかどうかはAIには分かりません。実際にあった事例として、米国の弁護士がChatGPTに判例を調べさせたところ、完全に架空の判例を6件も引用してしまい、裁判所で問題になったケースがあります。自信満々の文体なので見抜きにくいのが厄介です。業務でAIを使う際は、事実を含む情報は必ず裏を取る習慣をつけてください。

## スライド 12: 限界2 最新情報・計算・倫理（75秒 / 約625文字）

**【スライド内容】**
- 見出し: 生成AIの限界2: 最新情報・計算・倫理判断
- 3項目:
  - 最新情報: 学習データの時点で止まっている
  - 計算: 多桁の掛け算、論理パズルに弱い
  - 倫理判断: 善悪の価値判断は本質的にできない

**【ナレーション】**
限界の2つ目以降をまとめて見ていきましょう。まず「最新情報を知らない」点です。学習データには締め切りがあります。それ以降の出来事を知りません。昨日のニュースや最新の法改正を聞いても、正確に答えられないのはこのためです。検索連携機能もありますが、それはAI自身の知識ではなく外部ツールの力です。次に「計算が苦手」という点。AIは電卓ではなく、言葉の予測エンジンです。複雑な計算は苦手で、桁数が増えるほど間違える確率が上がります。ただし、OpenAIのo3-proのように「推論」に特化したモデルも登場し、論理的な問題への対応力は向上しつつあります。最後に「倫理判断ができない」点。AIは善悪の判断を本質的にはできません。RLHFで不適切な回答を抑制していますが、それはパターンであり判断ではありません。これらの限界はすべて「次の言葉を予測する」という根本的な仕組みに由来しています。
