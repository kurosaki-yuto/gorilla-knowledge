## スライド 3: AIエージェントとは？（60秒）

**【スライド内容】**
- 定義: 自分で考え、判断し、行動まで実行できるAIシステム
- 比較: 従来のAI vs AIエージェント

**【ナレーション】**
AIエージェントを一言で言うと、自分で考え、判断し、行動まで実行できるAIシステムです。従来のAIは人間が指示を出して、AIが1回答えて、また人間が次の指示を出すという流れでした。一方、AIエージェントは目標を伝えるだけで、AIが自分で計画を立て、自分で実行してくれます。エージェントとは代理人という意味。人間の代わりに自律的にタスクを進めてくれるわけです。

## スライド 4: AIエージェントの3つの特徴（40秒）

**【スライド内容】**
- 自律的に判断 / ツールを使える / 繰り返し改善

**【ナレーション】**
AIエージェントには3つの大きな特徴があります。1つ目は自律的に判断すること。次に何をすべきか自分で考えます。2つ目はツールを使えること。ウェブ検索、計算、ファイル操作など、外部のツールと連携して行動できます。3つ目は繰り返し改善すること。結果を見て、うまくいかなければ別の方法を試します。

**【スライド内容】**
- チャットボット: URL案内するだけ
- AIエージェント: 予約完了まで実行

**【ナレーション】**
従来のチャットボットとの違いを見てみましょう。たとえば「会議室を予約して」と頼んだとき、チャットボットは予約ページのURLを教えてくれるだけです。一方、AIエージェントは空き状況を確認し、最適な部屋を選び、予約完了まで全部やってくれます。1問1答から、目標達成まで自走するAIに進化したわけです。

## スライド 7: 身近なAIエージェントの例（40秒）

**【スライド内容】**
- GitHub Copilot Agent / Devin / Copilot Studio

**【ナレーション】**
すでに実用化されているAIエージェントを紹介します。GitHub Copilot Agentは、イシューからコード生成、テスト、プルリクエスト作成まで自動実行します。Devinはソフトウェア開発を一貫して対応できるAIエージェントです。Microsoft Copilot Studioは、社内システムと連携して業務フローを自動化できます。

## スライド 8: 従来AI vs AIエージェント（40秒）

**【スライド内容】**
- 比較テーブル: 動作 / ツール利用 / 判断力 / 代表例

**【ナレーション】**
従来のAIとAIエージェントを比較してみましょう。動作は1回の質問に1回回答から、目標達成まで自律的に動くへ。ツール利用はテキスト生成のみから、検索やAPI、ファイル操作も可能へ。判断力は人間が毎回指示から、状況を見て自分で次の行動を決めるへ。大きな進化ですね。

## スライド 9: 業務でどう使える？（50秒）

**【スライド内容】**
- メール対応 / リサーチ / データ処理 / カスタマー対応

**【ナレーション】**
では、業務ではどう使えるのか。メール対応では、受信内容を判断して返信下書きから送信まで。リサーチでは、テーマ設定から情報収集、レポート作成まで。データ処理では、ファイル取得から集計、報告書作成まで。カスタマー対応では、問い合わせの調査から回答、必要に応じたエスカレーション判断まで。指示1回で完了まで動いてくれるのがポイントです。

## スライド 10: AIエージェントの注意点（50秒）

**【スライド内容】**
- 権限管理 / 結果の確認 / 機密データに注意

**【ナレーション】**
AIエージェントを使うときの注意点を3つ覚えてください。1つ目、権限管理が重要。エージェントに与える権限は必要最小限に。何を操作できるか事前に確認しましょう。2つ目、結果の確認は必須。自律的に動くからこそ、最終成果は必ず人間がレビューしてください。3つ目、機密データに注意。エージェントがアクセスするデータの範囲を把握しておきましょう。
