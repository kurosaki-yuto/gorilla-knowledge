# 自社LMS 開発要件 & 助成金提出物ガイド

> 最終更新: 2026年3月19日

---

## 全体像

```
うち（訓練機関）
  └ LMSでコンテンツ提供 + 証拠書類を企業に渡す

企業（申請者）
  └ うちのLMSデータを使って労働局に助成金申請

都道府県労働局
  └ 審査して助成金を企業に支給
```

---

## Part 1: 自社LMSに必要なコア機能（最小5つ）

### 1. 動画プレイヤー（視聴ログ付き）

```
必須:
  ・動画の再生/停止
  ・視聴開始日時・完了日時の記録（DB保存）
  ・動画ごとの「標準学習時間」を表示（例: 20分）
  ・視聴完了の判定（最後まで再生したかどうか）

推奨:
  ・初回視聴時の早送り/スキップ禁止
  ・途中再開（前回の続きから再生）
```

### 2. テスト機能

```
必須:
  ・選択式テスト（3〜5問/コンテンツ）
  ・合格判定（正答率80%以上）
  ・受験日時・スコアの記録（DB保存）

※ テスト時間は助成金の時間にカウントされない
  → 時間記録は不要、合否だけ取ればOK
```

### 3. 修了判定 + 修了証PDF発行

```
修了条件:
  ・動画視聴が完了している
  ・確認テストに合格している
  → 両方満たしたら「修了」ステータスに変更

修了証PDFに記載する項目:
  ・受講者氏名
  ・コンテンツ名（コース名）
  ・標準学習時間
  ・修了日
  ・訓練機関名（うちの会社名）
  ・発行者の署名 or 印影

重要: 未修了の人には修了証が出せない制御が必須
```

### 4. 受講履歴のCSV出力

```
出力カラム:
  ・受講者名
  ・コンテンツID
  ・コンテンツ名
  ・標準学習時間（分）
  ・視聴開始日時
  ・視聴完了日時
  ・テスト合否
  ・修了ステータス（修了 / 未修了）
  ・修了日

→ 企業ごとにフィルタしてCSVダウンロードできること
```

### 5. 管理画面

```
コンテンツ管理:
  ・動画アップロード
  ・標準学習時間の設定
  ・テスト問題の登録

企業管理:
  ・企業の登録（契約情報）
  ・受講者の登録（企業に紐づく）

レポート:
  ・受講状況の一覧表示（企業別・受講者別）
  ・CSVダウンロードボタン
  ・修了証の一括/個別ダウンロード
```

## Part 2: うちのLMSから出力するもの（企業に渡すもの）

企業が労働局に提出するために、うちから渡す書類は **4つ**。

| # | 出力物 | 形式 | 内容 |
|---|--------|------|------|
| 1 | **受講履歴データ** | CSV | 誰が・いつ・何を・何分受講して・修了したか |
| 2 | **修了証** | PDF | 受講者ごとに1枚。修了者のみ発行 |
| 3 | **カリキュラム一覧** | PDF/CSV | コンテンツID・タイトル・標準学習時間の一覧 |
| 4 | **サービス利用料の領収書** | PDF | 企業が「全額負担した」ことの証拠 |

## Part 3: 助成金申請の流れ（何を・誰が・どこに・いつ）

### STEP 1: 計画届

| 項目 | 内容 |
|-----|-----|
| **誰が** | 企業（申請者） |
| **どこに** | 企業所在地の管轄 **都道府県労働局** |
| **いつ** | サービス契約開始の **1か月前まで** |

#### 提出書類

| 様式番号 | 書類名 | 作成者 |
|---------|--------|-------|
| 様式第1-1号 | 職業訓練実施計画届 | 企業（うちが記載例を提供） |
| 様式第3-2号 | 定額制サービスによる訓練に関する対象労働者一覧 | 企業 |
| 様式第14-1号 | 定額制サービスによる訓練に関する事業所確認票 | 企業 |
| — | 訓練カリキュラム・受講案内 | **うちが用意** |
| — | 定額制サービスの契約書 or 申込書 | **うちが用意** |

### STEP 2: 訓練実施中

| やること | 担当 |
|---------|------|
| LMSで受講記録を蓄積 | うち（自動） |
| 社員に受講させる（業務時間内・業務命令） | 企業 |
| 賃金を支払う（通常勤務扱い） | 企業 |

### STEP 3: 支給申請

| 項目 | 内容 |
|-----|-----|
| **誰が** | 企業（申請者） |
| **どこに** | 同じ **都道府県労働局** |
| **いつ** | サービス契約期間終了後 **2か月以内** |

#### 提出書類

| 様式番号 | 書類名 | 作成者 |
|---------|--------|-------|
| 様式第4-2号 | 支給申請書 | 企業 |
| 様式第6-3号 | 定額制サービスによる訓練に関する経費助成の内訳 | 企業 |
| 様式第8-5号 | 定額制サービスによる訓練実施結果報告書 | 企業（**うちのLMSデータを元に作成**） |
| — | LMSデータ（受講履歴CSV） | **うちのLMSから出力** |
| — | 修了証 | **うちのLMSから出力** |
| — | サービス利用料の領収書・振込明細 | **うちが発行** |

### STEP 4: 審査・支給

| 項目 | 内容 |
|-----|-----|
| **誰が** | 都道府県労働局 |
| **何を** | 提出書類を審査、要件を満たしているか確認 |
| **いつ** | 支給申請から通常1〜3か月後に支給 |

## Part 4: うちが事前に用意しておくべきもの

企業がスムーズに申請できるように、以下を準備しておく。

### 企業に渡すテンプレート・ガイド

- [ ] 訓練カリキュラム（コンテンツ一覧 + 標準学習時間）
- [ ] 受講案内（LMSログイン方法、受講手順、修了条件の説明）
- [ ] サービス契約書テンプレート
- [ ] 計画届の記載例（様式第1-1号の書き方サンプル）
- [ ] 助成金申請ガイド（「何をいつ出すか」の簡易マニュアル）

### LMSから出力できるようにしておくもの

- [ ] 受講履歴CSV（企業別フィルタ）
- [ ] 修了証PDF（個別 + 一括ダウンロード）
- [ ] カリキュラム一覧PDF
- [ ] 領収書PDF

## Part 5: データベース設計メモ（最小構成）

```
companies（企業）
  ├ id
  ├ name（企業名）
  ├ contract_start_date（契約開始日）
  ├ contract_end_date（契約終了日）
  └ plan（プラン）

users（受講者）
  ├ id
  ├ company_id → companies.id
  ├ name（氏名）
  └ email

contents（コンテンツ）
  ├ id
  ├ title（タイトル）
  ├ standard_duration_min（標準学習時間・分）
  ├ video_url（動画URL）
  └ level（レベル: 入門/実践/応用/管理者）

quizzes（テスト問題）
  ├ id
  ├ content_id → contents.id
  ├ question（問題文）
  ├ choices（選択肢・JSON）
  └ correct_answer（正解）

watch_logs（視聴ログ）
  ├ id
  ├ user_id → users.id
  ├ content_id → contents.id
  ├ started_at（視聴開始日時）
  ├ completed_at（視聴完了日時・NULL=未完了）
  └ is_completed（完了フラグ）

quiz_results（テスト結果）
  ├ id
  ├ user_id → users.id
  ├ content_id → contents.id
  ├ score（得点）
  ├ passed（合否）
  └ taken_at（受験日時）

completions（修了記録）
  ├ id
  ├ user_id → users.id
  ├ content_id → contents.id
  ├ completed_at（修了日）
  └ certificate_issued（修了証発行済フラグ）
```
